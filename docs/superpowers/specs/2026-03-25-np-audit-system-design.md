# 네이버 플레이스 진단 시스템 설계서

> 작성일: 2026-03-25
> 상태: 승인됨
> 목적: 소상공인 대상 네이버 플레이스 1차 진단 → 리포트 자동 생성 프로세스 구축

---

## 1. 시스템 개요

### 목적
네이버 플레이스 URL과 기본 정보를 입력받아 100점 만점 진단 리포트를 자동 생성한다.
박리다매 수주 모델의 핵심 영업 도구 + 유료 서비스 첫 산출물로 활용.

### 2단계 퍼널 구조

```
Lite (무료, 15분) → Full (유료, 첫 산출물) → 4주 코칭 (/np-onboard)
```

- **Lite**: 플레이스 URL만으로 빠르게 1장 진단. 영업 도구.
- **Full**: 에드로그 + 경쟁사 + 키워드 분석 포함 풀 진단서. 계약 후 첫 납품물.

### 커맨드 인터페이스

```
/np-audit lite   → Lite 모드 (URL + 기본 설문)
/np-audit full   → Full 모드 (URL + 에드로그 + 경쟁사 + 키워드)
/np-audit        → 모드 선택 프롬프트
```

## 2. 100점 만점 채점 체계

### S등급 — 50점 (매출 직결)

| # | 항목 | 배점 | 채점 기준 | 근거 |
|---|------|------|----------|------|
| S1 | 영수증 리뷰 & 별점 | 20점 | 리뷰 수: 0→0, ~100→4, ~300→8, ~500→12, 1000+→16점. 키워드 포함 리뷰 유도 장치(엽서/카드/이벤트) 유무: 4점 | 알고리즘이 키워드 단위로 리뷰를 읽음 (강의 기준). 4/6 별점 부활 대비 필수 |
| S2 | 플레이스 광고 | 15점 | 미집행→0, 집행 중→10, 키워드 최적화+A/B 테스트→15 | 광고 사용 시 매출 6배 (D-플레이스 리포트) |
| S3 | 예약 & 쿠폰 | 15점 | 예약 기능 ON→5, 알림받기 쿠폰→5, 톡톡 마케팅 메시지 활용→5 | 예약 사용 시 매출 6배, PV 3.1배 (D-플레이스 리포트). 쿠폰 사업자 94.2% 재활용 의향 |

## 3. 리포트 구조

### Lite 리포트 (1장, md 파일)

```
# [매장명] 네이버 플레이스 진단 리포트
> 진단일 | 플레이스 URL | 업종 | 상권

## 종합 점수: XX / 100점 [X등급]
- S등급(매출 직결): XX / 50
- A등급(노출 순위): XX / 30
- B등급(기본기): XX / 15
- 보너스(매출 부스터): XX / 5

## 즉시 개선 Top 3
1. [항목] — [현황] → [개선안] + [공식 데이터 근거]
2. ...
3. ...

## 잘하고 있는 것
- [강점 1]
- [강점 2]
- [강점 3]

## 개선 시 예상 효과
현재 XX점 → 기본 세팅 완료 시 예상 XX점 (X등급)
[핵심 키워드] 검색 시 상위 노출 가능권 진입 예상

---
브랜드라이즈 | 네이버플레이스 최적화 코칭
```

### Full 리포트 (풀 진단서, md + 노션)

```
# [매장명] 네이버 플레이스 풀 진단 보고서

## 1. 종합 점수 카드 (Lite와 동일)

## 2. 항목별 상세 진단 (13개 항목)
각 항목별:
- 현황 | 점수 | 판정 (OK / 주의 / 개선 필요 / 즉시 개선)
- 개선안 + 공식 데이터 근거
- 우선순위 (즉시 / 단기 / 중기)

## 3. 키워드 분석
- 현재 대표 키워드 (소스코드 추출)
- 경쟁사 대표 키워드 비교 (2~3곳)
- 마피아넷 연관 키워드 30개 + 추천 키워드 5개
- 자동완성 키워드 → 고객 니즈 해석
- N1(유사도) 기반 키워드 정합성 진단

## 4. 경쟁사 비교표
| 항목 | 우리 매장 | 경쟁사 A | 경쟁사 B | 경쟁사 C |
리뷰 수, 별점, 키워드, 사진, 예약, 쿠폰, 소식, 광고

## 5. N1/N2/N3 히든 지수 분석
- 에드로그 데이터 기반
- 키워드별 유사도/관련성/랭킹
- 순위 변동 원인 진단
- N1 낮음 → 키워드 정합성 개선 방향
- N2 낮음 → 체류시간/외부 유입 강화 방향

## 6. 4주 액션 플랜
- Week 1: S등급 즉시 세팅 (업주인증, 영업시간, 예약, 쿠폰, 광고)
- Week 2: A등급 + 키워드 최적화 (사진, 키워드, 소식, 블로그)
- Week 3: 리뷰 마케팅 + 외부 채널 (체험단, 인스타, 당근)
- Week 4: 최적화 + 루틴화 (N1/N2/N3 분석, 광고 A/B, 톡톡 메시지)

## 7. 예상 ROI
- 현재 점수 → 목표 점수 → 예상 순위 변화
- 공식 데이터 기반 예상 효과 (PV, 전환, 매출)
```

## 5. 데이터 수집 방법

### 자동 수집 가능 (플레이스 API/페이지)
- 매장명, 주소, 카테고리
- 리뷰 수
- 메뉴 정보 (개수, 사진 유무, 설명)
- 편의시설
- 길안내 정보
- AI 브리핑 활성화 여부

### 수동 확인 필요
- 영업시간 등록 여부
- 업주 인증 여부
- 예약/쿠폰/톡톡 활성화 여부
- 대표 키워드 (소스코드 "keywordlist")
- 소식/이벤트 발행 이력
- 플레이스 광고 집행 여부
- 블로그 체험단 운영 여부
- 외부 채널 연동 여부

### 유료 도구 필요
- 에드로그: N1/N2/N3 히든 지수, 키워드 순위 (월 2만원)
- 마피아넷: 연관 키워드 30개 분석 (무료)

## 6. 프로세스 플로우

### Lite 플로우 (15분)

```
1. 사장님으로부터 플레이스 URL 수령
2. /np-audit lite 실행
3. 플레이스 페이지에서 기본 데이터 수집 (자동 + 수동 확인)
4. 13개 항목 채점
5. Lite 리포트 생성 (md 파일)
6. 사장님에게 전달 (카톡/메일)
```

### Full 플로우 (1~2시간)

```
1. Lite 리포트 기반 계약 체결
2. 에드로그 등록 + 경쟁사 선정
3. 추가 데이터 수집 (에드로그, 키워드, 경쟁사)
4. /np-audit full 실행
5. Full 리포트 생성 (md + 노션)
6. 사장님에게 납품 + 설명 미팅
7. /np-onboard로 4주 코칭 시작
```

### 퍼널 구조

```
[리드 확보] → [Lite 진단 무료 제공] → [관심 확인]
     → [Full 진단 제안] → [계약] → [Full 납품]
     → [4주 코칭] → [월간 리테이너]
```

## 8. 기술 구현 범위

### 구현 대상
1. `/np-audit` 스킬 (lite/full 모드 분기)
2. 채점 엔진 (13개 항목 → 100점 환산)
3. 리포트 템플릿 (Lite 1장 / Full 풀 진단서)
4. 공식 데이터 멘트 자동 삽입 로직
5. `clients/{store-name}/np-audit.md` 자동 저장

### 향후 확장 (이번 범위 아님)
- 구글시트 연동 (다매장 일괄 관리)
- 노션 자동 페이지 생성
- 플레이스 API 자동 스크래핑
- 에드로그 API 연동 (있을 경우)
