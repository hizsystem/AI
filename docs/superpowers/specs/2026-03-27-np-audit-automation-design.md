## 1. 시스템 개요

### 핵심 변경 (v1 → v2)

| v1 (현재) | v2 (이번 구현) |
|----------|---------------|
| URL 받고 수동으로 전 항목 확인 | WebFetch 자동 크롤링 + 수동 질문 7개만 |
| md 리포트만 생성 | md 리포트 + 구글시트 (태스크 + 견적) 동시 생성 |
| 태스크 수동 정리 | 진단 결과 → 코칭 태스크 자동 변환 |
| 견적 케바케로 작성 | 패키지 기반 자동 산출 |
| 30분~1시간 소요 | 약 5분 소요 |

### 아키텍처

```
/np-audit lite URL
│
├── ① WebFetch: NP 페이지 크롤링 (자동 채점 ~45점)
├── ② 수동 질문 7개 (나머지 ~55점)
├── ③ 100점 채점 완료 + 등급 판정
│
├── 📄 md 리포트 → clients/{store}/np-audit.md
└── 📊 GAS webhook → 구글시트 4개 탭에 자동 분배
     ├── [매장 현황] 요약
     ├── [진단 상세] 항목별 점수
     ├── [태스크] 코칭 미션
     └── [견적] 패키지 추천 + 금액
```

### 로드맵

- **v1 (이번)**: Claude 스킬 + GAS 연동
- **v1.5 (바로 이후)**: 진단 완료 시 슬랙 알림 자동 발송 (GAS에 Slack Webhook 추가)
- **v2 (매장 30개+)**: 슬랙봇 or 웹 대시보드

## 2. 자동 크롤링 (WebFetch)

### 대상 URL
`m.place.naver.com/restaurant/{id}/home`

### 자동 수집 가능 데이터

| 항목 | 추출 위치 | 채점 반영 |
|------|----------|----------|
| 매장명 | 페이지 타이틀/헤더 | B3 (지역명 포함 여부) |
| 카테고리 | 업종 표시 영역 | B3 (카테고리 최적화) |
| 주소 | 주소 영역 | 기본정보 |
| 리뷰 수 | 리뷰 카운트 | S1 (리뷰 수 기준 점수) |
| AI 브리핑 텍스트 | AI 요약 영역 | B3 (AI 브리핑 ON/OFF) |
| 메뉴 목록 | 메뉴 탭 | A1 (메뉴별 사진 완비), 메뉴명 키워드 체크 |
| 메뉴 사진 유무 | 메뉴 이미지 태그 | A1 |
| 소개문 | 상세 소개 영역 | B2 (글자수 기준) |
| 영업시간 | 영업정보 | B1 |
| 편의시설 | 편의정보 영역 | B1 |
| 길안내 | 교통 정보 | B1 |
| 대표 키워드 | 소스코드 "keywordList" 파싱 | A2 (세팅 유무 + 지역명 포함) |

### 자동 채점 가능 항목 (~45점)

| 항목 | 배점 | 자동 채점 분량 |
|------|------|--------------|
| S1 리뷰 수 기준 | 16/20점 | 자동 |
| A1 메뉴 사진 완비 | 2/8점 | 자동 |
| A2 키워드 세팅+지역명 | 5/8점 | 자동 |
| B1 기본정보 완성도 | 6/6점 | 자동 |
| B2 소개문 글자수 | 5/5점 | 자동 |
| B3 AI브리핑+매장명 | 4/4점 | 자동 |

## 3. 수동 질문 (7개)

자동으로 못 잡는 항목만 순차적으로 질문.

| # | 질문 | 선택지 | 반영 항목 |
|---|------|--------|----------|
| Q1 | 대표 사진 퀄리티 — 먹고 싶은 느낌? | 상/중/하 | A1 (3점) |
| Q2 | 영상 소식 업로드되어 있나? | Y/N | A1 (3점) |
| Q3 | 예약 ON / 쿠폰 세팅 / 톡톡 활성화? | 각각 Y/N | S3 (15점) |
| Q4 | 플레이스 광고 집행 중? | 미집행/집행중/A/B테스트 | S2 (15점) |
| Q5 | 리뷰 가이드 엽서/이벤트 안내 있나? | Y/N | S1 (4점) |
| Q6 | 최근 30일 내 소식 발행? | Y/N + 이벤트 연동 Y/N | A3 (7점) |
| Q7 | 인스타/당근/블로그 연동? 커넥트? | 각각 Y/N | X1 (3점) + X2 (2점) |

수동 채점 분량: ~55점

## 5. 코칭 태스크 자동 생성

### 변환 규칙

"즉시 개선" 또는 "개선 필요" 판정 항목 → 태스크로 자동 변환.

| 진단 항목 | 판정 | 생성 태스크 | 우선순위 | 담당 |
|----------|------|-----------|---------|------|
| S1 리뷰 유도 없음 | 즉시 개선 | 영수증 리뷰 가이드 엽서 제작 + 배치 | Week 1 | 우리 |
| S2 광고 미집행 | 즉시 개선 | 플레이스 광고 세팅 (일 5,000원~) | Week 1 | 우리 |
| S3 예약 OFF | 즉시 개선 | 네이버 예약/톡톡/스마트콜 활성화 | Week 1 | 사장님 |
| S3 쿠폰 OFF | 즉시 개선 | 알림받기 쿠폰 세팅 | Week 1 | 사장님 |
| A1 사진 미흡 | 개선 필요 | 대표 사진 재촬영 (전문 or 가이드) | Week 2 | 사장님 |
| A1 영상 없음 | 개선 필요 | 5~10초 매장 영상 촬영 + 소식 업로드 | Week 2 | 사장님 |
| A2 키워드 미세팅 | 즉시 개선 | 대표 키워드 5개 세팅 (지역명 필수) | Week 1 | 우리 |
| A2 키워드 부적합 | 개선 필요 | 대표 키워드 교체 (에드로그 기반) | Week 2 | 우리 |
| A3 소식 없음 | 개선 필요 | 이벤트 소식 1개 발행 (쿠폰 연동) | Week 2 | 사장님 |
| A4 체험단 없음 | 개선 필요 | 블로그 체험단 4~5명 섭외 | Week 2 | 우리 |
| B2 소개문 부족 | 개선 필요 | 상세 소개문 1,500자 작성 | Week 1 | 우리 |
| B3 매장명 지역명 없음 | 개선 필요 | 매장명에 지역명 추가 | Week 1 | 사장님 |
| X1 외부 채널 없음 | 개선 필요 | 인스타/당근 비즈프로필 세팅 | Week 3 | 사장님 |
| X2 커넥트 미등록 | 개선 필요 | 네이버 커넥트 전환 검토 | Week 3 | 사장님 |

### 태스크 속성

```
{
  store: string,       // 매장명
  task: string,        // 태스크 내용
  priority: string,    // week1 / week2 / week3 / week4
  owner: string,       // "사장님" / "우리"
  status: string,      // "미완료" / "진행중" / "완료"
  created: date,       // 생성일
  deadline: date,      // 마감일 (priority 기준 자동 계산)
  source: string       // 어떤 진단 항목에서 생성됐는지 (S1, A2 등)
}
```

## 6. 패키지 견적 자동 산출

### 패키지 구조 (3-Tier)

| 패키지 | 대상 | 포함 내용 | 기간 |
|--------|------|----------|------|
| **세팅** | D~C등급 | 초기 세팅 풀 코칭 + Full 진단 | 4주 |
| **관리** | B등급 | 주간 미션 + 리뷰 전략 + 월간 리포트 | 월간 |
| **프리미엄** | A등급 or 요청 | 세팅 + 관리 + 체험단 + 광고 운영 | 월간 |

### 옵션 항목

| 옵션 | 조건 (자동 추가) |
|------|----------------|
| 체험단 N팀 | A4 "개선 필요" 판정 시 |
| 광고 세팅 | S2 "즉시 개선" 판정 시 |
| 광고 월 운영 | S2 집행 중이나 최적화 필요 시 |
| 사진 촬영 | A1 "개선 필요" 판정 시 |
| 소개문 작성 | B2 "개선 필요" 판정 시 |

### 산출 로직

```
1. 등급 → 기본 패키지 결정
2. 항목별 판정 → 필요한 옵션 자동 추가
3. [가격표] 탭에서 금액 참조
4. 합계 산출
```

### 가격표 탭 구조

| 항목코드 | 항목명 | 금액 | 비고 |
|---------|--------|------|------|
| PKG-SET | 세팅 패키지 | (미정) | 4주 |
| PKG-MNG | 관리 패키지 | (미정) | 월간 |
| PKG-PRM | 프리미엄 패키지 | (미정) | 월간 |
| OPT-BLOG | 체험단 5팀 | (미정) | 1회 |
| OPT-AD-SET | 광고 세팅 | (미정) | 1회 |
| OPT-AD-MNG | 광고 월 운영 | (미정) | 월간 |
| OPT-PHOTO | 사진 촬영 | (미정) | 1회 |
| OPT-INTRO | 소개문 작성 | (미정) | 1회 |

→ 가격은 운영하면서 확정. 시트에서 숫자만 바꾸면 전 견적 자동 반영.

## 7. 구글시트 구조

### 스프레드시트: NP 코칭 마스터시트 (5개 탭)

**[가격표] 탭**: 가격 기준 단일 소스. 여기만 수정하면 전체 견적 자동 반영.

**[매장 현황] 탭**: 전 매장 점수/등급 한눈에.
- 컬럼: 매장명, 업종, 상권, 총점, 등급, S점수, A점수, B점수, 보너스, 진단일, 추천패키지, 계약상태

**[진단 상세] 탭**: 13개 항목별 점수 원본.
- 컬럼: 매장명, S1점수, S1판정, S2점수, S2판정, ... X2점수, X2판정

**[태스크] 탭**: 매장별 코칭 미션 + 완료 체크.
- 컬럼: 매장명, 태스크, 우선순위, 담당, 상태, 생성일, 마감, 출처항목, 비고
- 필터: 매장별 / 주차별 / 미완료만 / 담당별

**[견적] 탭**: 패키지 추천 + 자동 산출 금액.
- 컬럼: 매장명, 등급, 추천패키지, 기본가, 옵션항목, 옵션가, 합계, 제안일, 상태
- 기본가/옵션가 = [가격표] 탭에서 VLOOKUP 자동 참조

## 8. GAS 웹훅 (doPost)

### 기존 자산
- `np-audit-gas.js`: 진단응답을 1개 시트에 저장하는 단순 구조

### 확장 구현
- 1개 엔드포인트(doPost)로 JSON 수신
- JSON 내 type 필드로 분기: `audit-summary`, `audit-detail`, `tasks`, `estimate`
- 각 type에 맞는 탭에 행 추가
- [견적] 탭은 금액 컬럼에 `=VLOOKUP(패키지코드, 가격표!A:B, 2, FALSE)` 수식 자동 삽입

## 9. `/np-audit` 스킬 업그레이드

### 실행 흐름

```
1. URL 파싱 → place ID 추출
2. WebFetch: m.place.naver.com/restaurant/{id}/home
3. HTML 파싱 → 자동 채점 데이터 추출
4. 자동 채점 실행 (S1 일부, A1 일부, A2, B1, B2, B3)
5. 수동 질문 7개 순차 진행
6. 전체 100점 채점 + 등급 판정
7. md 리포트 생성 → clients/{store}/np-audit.md 저장
8. JSON 구성 → GAS webhook 전송 → 구글시트 4개 탭 업데이트
9. 완료 메시지 + 요약
```

### 산출물 3종

1. **md 리포트** — 클라이언트 전달용 상세 진단서
2. **구글시트 데이터** — 내부 관리용 (현황 + 상세 + 태스크 + 견적)
3. **터미널 요약** — 즉시 확인용 (점수, 등급, 즉시 개선 Top 3, 추천 패키지)

## 10. 구현 범위

### 이번에 만들 것
1. `/np-audit` 스킬 v2 (WebFetch + 수동 질문 하이브리드)
2. 채점 엔진 (자동+수동 → 100점 합산)
3. 태스크 자동 생성 로직 (판정 → 태스크 변환 규칙)
4. 패키지 견적 자동 산출 로직 (등급 → 패키지 + 옵션)
5. GAS 확장 (4개 탭 분배 + VLOOKUP 수식)
6. 구글시트 5개 탭 구조 세팅
7. md 리포트 템플릿 (기존 spec 기반)

### 이번에 안 만들 것
- 슬랙 알림 (v1.5)
- 슬랙봇 (v2)
- 에드로그 연동 (API 없음, 수동 유지)
- Full 모드 자동화 (Lite 먼저 완성)
- 웹 대시보드
