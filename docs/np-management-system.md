# 네이버플레이스 관리 체계

> 작성: 2026-04-05
> 목적: 우리 액션의 효과를 숫자로 증명하는 관리 루틴
> 대상: 미례국밥 센텀점, 명동식당

---

## 핵심 원칙

**"매출이 올랐다"가 1번. 나머지는 근거 자료.**

증명의 신뢰도 순서:
1. **매출/방문** — 반박 불가 ("돈이 더 들어왔다")
2. **에드로그 데이터** — 제3자 도구 ("키워드 42위→9위")
3. **우리 진단표** — 보조 자료 ("28점→55점" = 참고용)

---

## 1. 증명 구조 — 3계층 (신뢰도 순)

### Layer 1: 매출/방문 지표 (최우선 — 사장님이 체감하는 것)

사장님이 "효과 있네"라고 느끼는 건 결국 **손님이 늘었는지**. 우리가 직접 POS를 못 보니까, 아래 프록시 지표를 잡는다.

**A. 사장님에게 직접 받는 것 (가장 강력)**

| 지표 | 왜 중요한지 | 받는 방법 |
|------|-----------|----------|
| 월 매출 (대략) | 반박 불가 | 월 1회 "지난달 매출 어떠셨어요?" |
| 일 평균 테이블 수 | 방문 객수 체감 | 주간 통화 시 "요즘 몇 테이블 정도?" |
| 체감 변화 | 정성적이지만 유효 | "저번 달보다 바빠지셨어요?" |

**B. 플레이스 통계에서 뽑는 것 (매출 프록시)**

| 지표 | 매출과의 관계 | 소스 | 주기 |
|------|-------------|------|------|
| **길찾기 클릭수** | 실제 방문 의도 = 매출 직결 | 플레이스 통계 (사장님 공유) | 주 1회 |
| **전화 클릭수** | 예약/문의 = 매출 직결 | 스마트콜 통계 | 주 1회 |
| **네이버 예약 건수** | 예약 = 확정 매출 | 예약 관리자 | 주 1회 |
| **알림받기 수** | 잠재 재방문 고객 DB | 플레이스 관리자 | 월 1회 |
| **월간 PV** | 관심도 → 방문 전환 | 플레이스 통계 | 월 1회 |

증명 방식: "길찾기 클릭 월 200→450 = 방문 의도 2.25배 증가"

**사장님에게 플레이스 통계 공유 요청하는 법:**
- 플레이스 관리자 > 통계 > 스크린샷 공유 (월 1회)
- 또는 관리자 권한 공유 받기 (이상적)

### Layer 2: 키워드 순위 + 에드로그 (제3자 검증)

에드로그는 우리 도구가 아니라 **제3자 분석 플랫폼**이라 신뢰도 있음.

```
[매장별 추적 키워드 — 최대 5개]

미례국밥 센텀점:
  1. 센텀 맛집 (14,230건) — 현재 9위
  2. 재송동 맛집 (4,460건) — 현재 3위
  3. 센텀국밥 (460건) — 현재 1위
  4. 센텀 돼지국밥 (300건) — 현재 2위
  5. 센텀점심 (460건) — 현재 5위

명동식당:
  (에드로그 등록 후 키워드 선정 필요)
```

증명 방식: "'센텀 맛집' 월 14,230명이 검색하는 키워드에서 42위→9위. 이건 에드로그 데이터입니다."

| 지표 | 측정 소스 | 주기 | 증명 예시 |
|------|----------|------|----------|
| 목표 키워드 순위 | 에드로그 | 주 1회 | "센텀 맛집 42위→9위 (2주)" |
| N1 (유사도) | 에드로그 | 주 1회 | "대표키워드 교체 후 0.369→0.45" |
| N2 (관련성) | 에드로그 | 주 1회 | "인스타 광고 시작 후 0.490→0.51" |
| 채널별 유입 | 에드로그 유입 통계 | 월 1회 | "인스타 유입 0→일 15건" |

### Layer 3: 우리 진단표 (보조 — 과정 설명용)

`/np-audit` 진단 점수는 **"우리가 뭘 했고 앞으로 뭘 할 건지" 설명하는 용도**로 쓴다. 증명이 아니라 로드맵.

| 매장 | 최초 진단 | 점수 | 1개월 후 목표 | 3개월 후 목표 |
|------|----------|------|-------------|-------------|
| 미례국밥 센텀점 | 2026-03-28 | 39점 D등급 | 60점 B등급 | 75점+ A등급 |
| 명동식당 | 2026-03-28 | 28점 D등급 | 55점 C등급 | 70점+ B등급 |

활용: "현재 13개 항목 중 8개가 미세팅. 이걸 하나씩 세팅하면 점수가 올라가고, 점수가 올라가면 순위가 올라가고, 순위가 올라가면 방문이 늘어납니다."

## 2. 주간 루틴 — 매주 월요일

### 데이터 수집 (10분)

| # | 할 일 | 소스 | 기록 위치 |
|---|------|------|----------|
| 1 | 에드로그 순위 체크 스크린샷 | 에드로그 | 주간 기록 시트 |
| 2 | 에드로그 히든 지수 (N1/N2) 스크린샷 | 에드로그 | 주간 기록 시트 |
| 3 | 플레이스 방문자리뷰 수 확인 | 플레이스 직접 | 주간 기록 시트 |
| 4 | 플레이스 블로그리뷰 수 확인 | 플레이스 직접 | 주간 기록 시트 |
| 5 | 플레이스 저장수 확인 | 에드로그 | 주간 기록 시트 |

### 분석 + 액션 결정 (5분)

```
주간 체크 질문 3개:

1. 목표 키워드 순위가 올랐나? 내렸나? → 왜?
2. 지난주 액션(체험단/광고/세팅)의 효과가 숫자에 보이나?
3. 이번 주 우선 액션은?
```

### 기록 형식

```markdown
## W{n} — {날짜}

### 스냅샷
| 지표 | 미례국밥 | 전주 | 명동식당 | 전주 |
|------|---------|------|---------|------|
| 핵심 키워드 순위 | | | | |
| N1 | | | | |
| N2 | | | | |
| 방문자 리뷰 | | | | |
| 블로그 리뷰 | | | | |
| 저장수 | | | | |

### 지난주 액션 → 결과
- 액션: {한 일}
- 결과: {숫자 변화}

### 이번 주 액션
- [ ] {할 일} (담당: 우리/사장님)
```

## 3. 월간 리포트 — 매월 말

### 구조: 매출 먼저, 근거 다음

```markdown
# {매장명} 월간 성과 리포트 — {월}

## 한줄 요약
"{월} 한 달간 {매출/방문 관련 핵심 성과 1문장}"

## 1. 매출/방문 변화 (사장님이 체감하는 것)

| 지표 | 지난달 | 이번달 | 변화 |
|------|-------|-------|------|
| 월 매출 (대략) | | | |
| 일 평균 테이블 | | | |
| 길찾기 클릭 | | | +N% |
| 전화 클릭 | | | +N건 |
| 예약 건수 | | | +N건 |
| 플레이스 PV | | | +N% |

## 2. 왜 올랐나 — 키워드 순위 변화 (에드로그)

| 키워드 | 월 검색량 | 지난달 순위 | 이번달 순위 | 변화 |
|--------|----------|-----------|-----------|------|
| | | | | +N계단 |

"'{키워드}' 월 {N}명이 검색합니다. 여기서 {N}위→{N}위로 올라갔습니다."
→ 이건 에드로그(제3자 분석 도구) 데이터입니다.

## 3. 우리가 한 일 → 숫자 변화 연결

| 우리가 한 일 | 언제 | 어떤 숫자가 바뀌었나 |
|-------------|------|-------------------|
| 대표키워드 교체 | W1 | N1 0.369→0.45, 키워드 정합성 개선 |
| 체험단 5명 | W2 | 블로그 44→49, 해당 키워드 노출 |
| 광고 세팅 | W1 | 길찾기 클릭 +30%, PV +40% |
| 리뷰 가이드 배치 | W1 | 주간 리뷰 +15건 |

## 4. 다음 달 계획
- {우선순위 1} → 예상 효과
- {우선순위 2} → 예상 효과
```

### 리포트의 설득 구조

```
사장님이 보는 순서:

"매출이 올랐어요?" → 1번에서 바로 확인
     ↓
"왜 올랐는데?" → 키워드 순위 데이터 (에드로그 = 제3자)
     ↓
"너네가 뭘 했는데?" → 액션→결과 연결
     ↓
"다음 달엔?" → 계획
```

### 증명력을 높이는 3가지 장치

**1. 에드로그 스크린샷 첨부**: 우리 말이 아니라 데이터 화면 그대로. "이 날 이 수치였다"의 증거.

**2. 액션 타임라인**: "이 날 이것을 했다 → 그 다음 주에 이 숫자가 바뀌었다"의 인과관계.

**3. 사장님 체감과 데이터 연결**: "바빠지셨다고 하셨죠? 플레이스 길찾기 클릭이 +45% 올랐습니다. 이게 이유입니다."

## 5. 도구 체계

| 도구 | 용도 | 비용 | 주기 |
|------|------|------|------|
| 에드로그 | 순위/N1/N2/히든키워드/유입통계 | 월 2만원 (1계정 전체) | 주 1회 |
| `/np-audit` | 100점 진단 Before/After | 무료 (Claude) | 월 1회 재진단 |
| `/np-keyword` | 키워드 심층 분석 | 무료 (Claude) | 월 1~2회 |
| `/np-report` | 월간 리포트 자동 생성 | 무료 (Claude) | 월 1회 |
| 구글시트 마스터시트 | 주간 데이터 누적 | 무료 | 주 1회 입력 |

### 주간 데이터 기록 시트 구조

마스터시트에 "주간 추적" 탭 추가:

| 날짜 | 매장 | 키워드 | 순위 | N1 | N2 | 방문리뷰 | 블로그 | 저장수 | 액션메모 |
|------|------|--------|------|-----|-----|---------|-------|-------|---------|

이 데이터가 쌓이면 `/np-report`에서 자동으로 Before/After 표를 생성할 수 있음.
