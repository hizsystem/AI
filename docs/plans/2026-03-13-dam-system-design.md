# MarketingOS 디지털 에셋 관리(DAM) 체계 설계

> 작성일: 2026-03-13
> 상태: 설계 완료, 실행 대기

## 1. 배경

프로젝트 산출물(전략서, 제안서, 디자인, 이미지, 계약서 등)이 로컬, Synology NAS, Notion, Google Sheets에 흩어져 있어 찾기 어려운 상황. 체계적인 디지털 에셋 관리 시스템이 필요.

## 2. 접근 방식: A+C 하이브리드

- **Notion** = 중앙 인덱스 (검색/필터/태그)
- **Synology NAS** = 바이너리 파일 창고 (디자인, 이미지, 영상, PDF)
- **AI/ Git 레포** = 텍스트 산출물 (전략서, 제안서, 보고서 — 버전관리 + Claude Code 활용)
- **Google Sheets** = 재무/데이터 (기존 유지)

## 3. 아키텍처

```
┌─────────────────────────────────────────────────────────┐
│                  Notion (중앙 인덱스)                     │
│  ┌─────────────┐  ┌──────────────┐  ┌───────────────┐  │
│  │ 에셋 카탈로그 │  │ 프로젝트 허브  │  │ 클라이언트 DB │  │
│  │  (마스터 DB) │  │ (프로젝트별)  │  │  (기존 활용)  │  │
│  └──────┬──────┘  └──────┬───────┘  └───────┬───────┘  │
│         └───────────────┼───────────────────┘          │
│                  Relation으로 연결                       │
└─────────────────────┬───────────────────────────────────┘
                      │ 링크 참조
          ┌───────────┼───────────┐
          ▼           ▼           ▼
    ┌──────────┐ ┌──────────┐ ┌──────────────┐
    │ Synology │ │  AI/     │ │ Google       │
    │ NAS      │ │  Git레포  │ │ Sheets       │
    │ 바이너리  │ │ 텍스트   │ │ 재무/데이터   │
    └──────────┘ └──────────┘ └──────────────┘
```

MarketingOS/

│   └── templates/

│   └── {client-name}/
│       └── {project-name}/
│           ├── 01-brief/
│           ├── 02-research/
│           ├── 03-strategy/
│           ├── 04-creative/
│           │   ├── working/       # 작업중 파일

## 6. 파일 네이밍 컨벤션

```
{클라이언트}_{에셋유형}_{설명}_{버전}_{날짜}.{확장자}
```

예시:
- `samscabin_proposal_브랜딩패키지_v2_20260312.pdf`
- `venturium_sns_인스타피드_w03_20260310.png`
- `withrun_contract_SNS운영_v1_20260304.pdf`

에셋 유형 약어:
| 약어 | 의미 |
|------|------|
| proposal | 제안서 |
| strategy | 전략 문서 |
| report | 보고서 |
| sns | SNS 콘텐츠 |
| design | 디자인 파일 |
| logo | 로고 |
| contract | 계약서 |
| brief | 브리프 |
| photo | 사진 |
| video | 영상 |

## 7. Notion 데이터베이스 설계

### DB 1: 에셋 카탈로그 (마스터)

| 속성 | 타입 | 설명 |
|------|------|------|
| 에셋명 | Title | 에셋 이름 |
| 클라이언트 | Relation → 클라이언트 DB | 소속 클라이언트 |
| 프로젝트 | Relation → 프로젝트 DB | 소속 프로젝트 |
| 에셋 유형 | Select | proposal/strategy/sns/design 등 |
| 카테고리 | Select | 문서/디자인/영상/사진/기타 |
| 상태 | Select | 작업중/검토중/최종/아카이브 |
| 저장 위치 | Select | NAS/AI레포/Google Sheets |
| 파일 경로 | URL or Text | 실제 파일 위치 |
| NAS 링크 | URL | Synology 공유 링크 |
| 태그 | Multi-select | 자유 태그 |
| 생성일 | Date | 에셋 생성 날짜 |
| 버전 | Text | 현재 버전 |
| 메모 | Text | 특이사항 |

### DB 2: 클라이언트 DB

| 속성 | 타입 | 설명 |
|------|------|------|
| 클라이언트명 | Title | 회사/브랜드 이름 |
| 상태 | Select | 활성/완료/대기 |
| 산업 | Select | F&B/테크/뷰티 등 |
| 에셋 | Relation → 에셋 카탈로그 | 모든 에셋 |
| NAS 폴더 | URL | NAS 경로 |
| AI/ 폴더 | Text | Git 레포 경로 |

### DB 3: 프로젝트 DB

| 속성 | 타입 | 설명 |
|------|------|------|
| 프로젝트명 | Title | 프로젝트 이름 |
| 클라이언트 | Relation → 클라이언트 DB | 소속 클라이언트 |
| 타입 | Select | 브랜딩/SNS운영/퍼포먼스/GTM |
| 기간 | Date range | 프로젝트 기간 |
| 상태 | Select | 진행중/완료/보류 |
| 에셋 | Relation → 에셋 카탈로그 | 프로젝트 산출물 |

### Notion 뷰

- 마스터 뷰: 전체 에셋을 클라이언트별로 그룹화
- 검색 뷰: 태그/유형/카테고리로 필터
- NAS 파일 뷰: 저장위치=NAS인 것만
- AI레포 파일 뷰: 저장위치=AI레포인 것만
- 아카이브 뷰: 상태=아카이브인 것만
- 최근 추가 뷰: 생성일 내림차순

## 8. 운영 워크플로우

### 새 프로젝트 시작 시
1. Notion "프로젝트 DB"에 새 프로젝트 생성
2. NAS에 클라이언트/프로젝트 폴더 생성 (템플릿 복사)
3. AI/ 레포에 클라이언트 폴더 생성 (/new-client 활용)
4. 각 산출물 완성 시마다 에셋 카탈로그에 등록

### 산출물 완성 시 (5분 루틴)
1. 파일을 올바른 위치에 저장 (텍스트→AI/, 바이너리→NAS)
2. 파일명을 네이밍 컨벤션에 맞게 지정
3. Notion 에셋 카탈로그에 1행 추가

### 프로젝트 완료 시
1. 모든 에셋 상태를 "최종" 또는 "아카이브"로 변경
2. NAS: 완료 폴더를 _archive/2026/{client}/로 이동
3. Notion: 프로젝트 상태를 "완료"로 변경

### 월 1회 점검
1. "상태=작업중" 에셋 리뷰
2. NAS 미등록 파일 확인
3. 태그 정리

## 9. 마이그레이션 계획

### Phase 1: 체계 구축 (Day 1)
- NAS에 MarketingOS/ 폴더 구조 생성
- Notion에 3개 DB 생성
- AI/ 레포에 _asset-index.md 템플릿 추가
- 네이밍 컨벤션 가이드 작성

### Phase 2: 활성 프로젝트 우선 (Week 1)
- 현재 진행 중인 클라이언트 먼저 정리 (sams-cabin, withrun, venturium-sns)
- 핵심 에셋만 Notion 카탈로그에 등록

### Phase 3: 과거 프로젝트 아카이브 (Week 2~3)
- 완료된 프로젝트 아카이브
- 재사용 가능한 에셋만 카탈로그에 등록

### Phase 4: 새 프로젝트부터 풀 적용 (Week 3~)
- 새 프로젝트는 처음부터 DAM 워크플로우 적용
- 2~3개 프로젝트 운영 후 개선점 파악
