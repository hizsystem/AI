# 디지털 에셋 아카이빙 플랜 (요약)

> MarketingOS | 2026.03.16

---

## 현황 및 문제

클라이언트 산출물(전략서, 제안서, 디자인, 계약서 등)이 4곳에 분산되어 있어 검색·재활용이 어려움.

| 저장소 | 현재 용도 | 문제 |
|--------|----------|------|
| 로컬(Mac) | 작업 중 파일 | 백업 없음, 검색 불가 |
| Synology NAS | 디자인, 영상, PDF | 폴더 구조 비일관 |
| Notion | 프로젝트 관리 | 에셋 인덱스 없음 |
| Google Sheets | 재무 데이터 | 문제 없음 (유지) |

---

## 해결 방안: 3계층 하이브리드

기존 무료 도구를 활용한 역할 분리 구조.

```
Notion (검색·인덱스)  ←  중앙 카탈로그, 태그/필터로 에셋 검색
        │
    링크 참조
        │
   ┌────┴────┐
   ▼         ▼
 NAS       Git 레포
(바이너리)  (텍스트)
디자인·영상  전략서·제안서
PDF·이미지  보고서·콘텐츠
```

**핵심 원리**: 파일은 유형별 최적 저장소에 보관하고, Notion이 "어디에 뭐가 있는지"를 관리.

---

## 비용

**추가 비용: 0원**

| 도구 | 비용 | 비고 |
|------|------|------|
| Notion | 기존 사용 중 (무료/Pro) | 인덱스 DB 추가만 |
| Synology NAS | 기존 보유 | 폴더 구조 정리만 |
| Git(GitHub) | 기존 사용 중 | 텍스트 파일 버전 관리 |

---

## 기대 효과

| 항목 | Before | After |
|------|--------|-------|
| 에셋 검색 시간 | 5~15분 (폴더 뒤지기) | 30초 이내 (Notion 검색) |
| 재활용률 | 낮음 (찾을 수 없어서) | 태그 기반 유사 에셋 발견 |
| 백업 안정성 | 로컬만 (유실 위험) | NAS(자동 백업) + Git(버전 이력) |
| 온보딩 | 구두 전달 | 폴더 구조 + 네이밍 규칙으로 표준화 |

---

## 실행 계획 (4주)

| 주차 | 내용 | 소요 시간 |
|------|------|----------|
| 1주 | NAS 폴더 구조 정리 + Notion DB 3개 생성 + 네이밍 규칙 확정 | 반나절 |
| 1~2주 | 진행 중인 클라이언트(2~3개) 에셋 먼저 정리 | 틈틈이 |
| 2~3주 | 완료 프로젝트 아카이브 이동 | 틈틈이 |
| 3주~ | 신규 프로젝트부터 풀 적용, 운영하며 개선 | 일상 루틴화 |

산출물 완성 시마다 **5분 루틴**(저장→네이밍→Notion 1행 추가)으로 유지.
