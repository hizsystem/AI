# 마스터시트 시스템 설계서

> 작성일: 2026-02-23
> 상태: 승인됨
> 회계팀 마감: 2026-02-27

---

## 1. 개요

3팀이 운영하는 다수 프로젝트의 재무/일정 관리를 위한 Google Sheets 마스터시트 시스템.
전용 에이전트 `spreadsheet-architect`가 시트 구조 설계, API 자동 생성, 데이터 관리를 담당한다.

### 회계팀 요청 사항

1. **2026년 프로젝트별 예상 매출 발생 내역** (기한: ~2/27)
   - 예상 매출 발생 시기 + 금액 (세금계산서 발행 기준)
   - 프로젝트별 마스터시트에 기입, 지속 업데이트/관리

2. **2026년 프로젝트별 예상 지출 발생 내역** (기한: ~2/27)
   - 종료시점까지 월별 실제 지출 예상 비용
   - 프로젝트별 마스터시트에 기입, 지속 업데이트/관리

## 2. 에이전트: spreadsheet-architect

| 항목 | 내용 |
|------|------|
| 파일 | `.claude/agents/spreadsheet-architect.md` |
| 이름 | (📑) 스프레드시트 아키텍트 |
| 역할 | 마스터시트 구조 설계 + Google Sheets API 자동 생성 + 재무 데이터 관리 |

### 핵심 역량

1. **시트 구조 설계** — 탭 구성, 열/행 구조, 수식 설계
2. **Google Sheets API 실행** — Python + gspread로 시트 자동 생성/업데이트
3. **재무 데이터 관리** — 매출(예상/실현), 지출(예산/실지출), 수익 계산
4. **템플릿 표준화** — 프로젝트 성격 무관한 공통 구조 유지

### 에이전트 원칙

- **회계팀 친화적** — 세금계산서 기준, 월별 집계, 카테고리별 분류
- **일관성 우선** — 모든 프로젝트 시트가 같은 기본 탭 구조
- **자동 연동** — 프로젝트 시트 → 팀 총괄 시트 자동 집계 (IMPORTRANGE)
- **확장 가능** — 프로젝트 추가 시 템플릿 복제로 즉시 대응

## 3. 프로젝트 마스터시트 구조

프로젝트마다 1개의 Google Sheets 파일을 생성한다.

### 필수 탭 (공통)

#### 탭 1: 종합-요약-색인

| 영역 | 내용 |
|------|------|
| 프로젝트 개요 | 프로젝트명, 클라이언트, 담당팀, 계약기간, 계약금액 |
| 핵심 수치 요약 | 총 예상매출, 총 예산, 현재 실매출, 현재 실지출, 수익률 |
| 진행 상태 | 전체 TASK 수, 완료/진행중/미착수, 진행률(%) |
| 색인 | 각 탭 하이퍼링크 |

#### 탭 2: 통합 TASK / 스케줄

| 열 | 설명 |
|----|------|
| TASK ID | 자동 번호 (T-001~) |
| 카테고리 | 기획/디자인/개발/마케팅/미팅 등 |
| TASK명 | 세부 업무 |
| 담당자 | 팀원 또는 에이전트 |
| 시작일 / 마감일 | 날짜 |
| 상태 | 미착수/진행중/완료/보류 |
| 우선순위 | 상/중/하 |
| 비고 | 메모, 링크 |

#### 탭 3: 2026 예상 매출

| 열 | 설명 |
|----|------|
| 항목 | 매출 발생 근거 (1차 계약금, 중간 정산, 잔금 등) |
| 세금계산서 발행일 | 회계팀 요청 기준 |
| 금액 (공급가) | 부가세 별도 |
| 금액 (VAT 포함) | 총액 |
| 1월~12월 | 해당 월에 금액 배치 |
| 월별 합계 | SUM 수식 |
| 연간 합계 | 총 예상 매출 |

#### 탭 4: 2026 예상 지출

| 열 | 설명 |
|----|------|
| 카테고리 | 인건비/외주비/광고비/툴비용/기타 |
| 항목 | 세부 지출 내역 |
| 1월~12월 | 월별 예상 지출 |
| 카테고리별 소계 | 카테고리별 SUM |
| 월별 합계 | 월별 총 지출 |
| 연간 합계 | 총 예상 지출 |

### 선택 탭 (프로젝트 성격에 따라)

| 탭 | 용도 | 해당 유형 |
|----|------|----------|
| 콘텐츠 캘린더 | SNS/블로그 발행 일정 | 마케팅/콘텐츠 |
| 캠페인 성과 | 광고 KPI 트래킹 | 퍼포먼스 마케팅 |
| 커뮤니케이션 로그 | 미팅/이메일 이력 | 외부 클라이언트 |
| 산출물 체크리스트 | 납품물 목록 + 상태 | 에이전시형 |

## 4. 팀 총괄 마스터시트 구조

3팀 전체를 한눈에 보는 1개의 Google Sheets 파일.

### 탭 1: 대시보드

| 영역 | 내용 |
|------|------|
| 팀별 프로젝트 현황 | 팀명, 프로젝트 수, 진행중/완료 |
| 월별 전체 수익 요약 | 매출 - 지출 = 수익 (월별) |
| 연간 누적 수익 | 차트용 데이터 |

### 탭 2: 프로젝트별 월별 총지출

- 행: 프로젝트별 1행
- 열: 1월~12월 + 연간합계
- 셀: 각 프로젝트 시트 "예상 지출" 탭 월별 합계를 IMPORTRANGE로 참조
- 팀별 소계 행으로 그룹핑

### 탭 3: 팀 전체 월별 총지출

- 행: 카테고리별 (인건비/외주비/광고비/툴비용/기타)
- 열: 1월~12월 + 연간합계
- 셀: 탭 2 팀별 소계 집계
- 전체 합계: 3팀 합산

### 탭 4: 예상 매출 vs 지출 — 월별 수익

- 매출 행: 프로젝트별 월별 예상 매출
- 지출 행: 프로젝트별 월별 예상 지출
- 수익 행: 매출 - 지출 (프로젝트별)
- 팀별 수익 소계
- 전체 수익: 3팀 합산 월별 수익
- 누적 수익: 1월부터 누적 합산

## 5. 데이터 연동

```
[프로젝트A 마스터시트] ──┐
[프로젝트B 마스터시트] ──┼──→ [팀 총괄 마스터시트]
[프로젝트C 마스터시트] ──┘
```

- 방식: Google Sheets `IMPORTRANGE` 수식
- 참조 대상: 각 프로젝트 시트의 월별 합계 셀만 (성능 최적화)
- 프로젝트 추가 시: 총괄 시트에 행 추가 + IMPORTRANGE 수식 삽입 (에이전트 자동화)

## 6. 기술 스택

| 항목 | 선택 |
|------|------|

| 실행 환경 | Claude Code Bash (Python 스크립트) |
| 시트 간 연동 | IMPORTRANGE 수식 |

## 7. 구현 순서

1. GCP 프로젝트 생성 + Sheets API 활성화 + Service Account 키 발급
2. `spreadsheet-architect` 에이전트 `.md` 파일 생성
3. Python 시트 생성 스크립트 작성 (프로젝트 템플릿)
4. 프로젝트 마스터시트 1개 생성 (테스트)
5. 팀 총괄 마스터시트 생성
6. IMPORTRANGE 연동 설정
7. 회계팀 요청 데이터 입력 (매출/지출)
