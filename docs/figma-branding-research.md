# Figma 브랜딩/마케팅 활용 종합 리서치

리서치 일자: 2026-03-15

## 1. 주요 활용 영역

### 1.1 브랜드 아이덴티티 (로고, 컬러, 타이포)

- **Figma Draw** (2025~): 벡터 브러시, 다이나믹 스트로크, 노이즈/텍스처, 텍스트 온 패스 -- 뱃지/로고를 Figma 안에서 제작 가능
- **Vectorize** (AI): 래스터 이미지/손 그림을 편집 가능한 벡터로 변환
- **Figma Variables**: 3계층 토큰 구조 (brand, alias, mapped collections)로 컬러 시스템 관리
- 텍스트 스타일을 팀 라이브러리로 공유하여 조직 전체 타이포그래피 통일

### 1.2 브랜드 가이드라인 / 브랜드북

- **AI Brand Guidelines Generator**: 브랜드 성격/가치/비주얼 방향을 자연어로 입력하면 컬러, 타이포, 레이아웃, 이미지, 보이스 규칙 자동 생성
- 커뮤니티 템플릿:
  - Brand Design Kit v6 -- 가이드라인, 피치덱, 이메일 등 올인원
  - Minimal Brand Identity Guidelines
  - Brand Guide Template
  - Branding Identity Mockups

### 1.3 마케팅 콜래트럴

- Figma Community: 60+ 마케팅 템플릿, 1000+ SNS 템플릿
- **Figma Buzz** (Config 2025): 비디자이너가 온브랜드 SNS/광고/이벤트 소재를 직접 제작
- **Figma Sites**: 랜딩 페이지, 이벤트 사이트, 캠페인 페이지를 코드 없이 퍼블리싱

### 1.4 캠페인 에셋 생산

- Figma Buzz AI 기능:
  - OpenAI gpt-image-1 기반 이미지 생성
  - 배경 제거, 이미지 결합, 해상도 향상
  - 텍스트 리라이팅, 축약, 번역 원클릭

### 1.5 디자인 시스템

- **Extended Collections** (2025): 하나의 코어 시스템으로 멀티 브랜드 지원
- 코어 토큰(접근성, 그리드) + 브랜드별 오버레이로 브랜드 고유값 적용
- Figma Buzz가 디자인 시스템에 직접 연결되어 일관성 보장

## 3. 워크플로우 최적화

### 3.1 컴포넌트/배리언트
- 원자 단위 컴포넌트 -> 조합 -> 큰 컴포넌트
- 배리언트로 상태 관리 (hover, active, disabled, 브랜드 A vs B)
- 컴포넌트 프로퍼티 (boolean, text, instance swap)로 유연성 확보

### 3.2 오토 레이아웃
- 8포인트 그리드 기반 패딩/간격
- 중첩 오토 레이아웃으로 복잡한 반응형 구조 대응
- 카드, 배너, SNS 템플릿에 적용하여 콘텐츠 길이 변화 자동 대응

### 3.3 디자인 토큰 3계층
```
Brand Tokens (기본값: 브랜드 고유 색상, 폰트)
  -> Alias Tokens (의미 부여: primary, secondary, success, error)
       -> Component Tokens (적용: button-bg, card-border, input-focus)
```
- Extended Collections로 멀티 브랜드 운영
- 수개월 주기로 토큰 시스템 리뷰

### 3.4 팀 라이브러리 구성
- 디자인 시스템용 별도 팀/프로젝트 구조
- Foundation (컬러, 타이포, 간격) / Components (버튼, 카드) / Templates (마케팅 레이아웃)
- 명확한 네이밍 컨벤션 + 문서화 페이지

### 3.5 브랜칭
- 캠페인 배리에이션 작업 시 브랜치 활용
- 네이밍: campaign-summer2026/variant-A
- 정기 머지로 메인 파일 최신 유지
- 같은 요소에 대한 동시 편집 시 수동 충돌 해결 필요

## 4. Figma + AI (2025-2026)

### 4.1 네이티브 AI 기능
- Vectorize: 래스터 -> 벡터 변환
- Expand Image: 하나의 제품 사진 -> 여러 광고 사이즈
- Erase / Isolate Object: 배경 제거, 오브젝트 분리
- AI Brand Guidelines Generator: 자연어 -> 브랜드 시스템
- AI Landing Page Generator: 프롬프트 -> 랜딩 페이지
- Figma Make: 프롬프트 -> 프로토타입/웹앱
- AI 구독: 월간 크레딧 + 종량제 (Organization/Enterprise 즉시, Professional 2026.5부터)

### 4.2 MCP 연동
- Figma MCP 서버: AI 에이전트(Claude Code, Cursor, Copilot)가 직접 접근
  - 노드 트리, 배리언트, 레이아웃 제약, 디자인 토큰, 에셋 참조 읽기
  - 디자인 토큰 적용/도입 위치 식별
  - 브랜드 사양에 맞는 코드 생성
- ServiceNow: MCP로 UI 구현 시간 80%+ 단축
- Figma 공식: "디자인 시스템이 AI 워크플로우의 생산성 계수가 된다"

### 4.3 업계 동향
- 78% 디자이너가 AI 효율성 향상 보고
- 2년 내 대부분의 디자인 팀이 AI 플러그인을 레이어/컴포넌트와 동급으로 취급 예측
- 풀스택 AI 워크플로우: 디자인 + 유저플로우 + 브랜딩 가이드라인 + 프로덕션 인터페이스 통합 생성

## 5. Figma vs 대안

### Figma 강점
- 실시간 협업
- 컴포넌트/디자인 시스템 관리
- 마케팅 콜래트럴 (디지털)
- AI 기능 (Buzz, Sites, Make, Vectorize)
- 커뮤니티 템플릿/플러그인

### Adobe가 필요한 영역
- CMYK 색상 관리 (인쇄)
- 복잡한 벡터 일러스트 (Illustrator)
- 다페이지 인쇄물 (InDesign)
- 정밀 타이포그래피 (Illustrator)
- 블리드, 크롭 마크, ICC 프로파일

### 하이브리드 워크플로우
1. 로고: Illustrator(정밀) -> Figma(시스템 통합)
2. 브랜드 가이드: Figma(디지털) + InDesign(인쇄용)
3. 마케팅 소재: Figma Buzz(디지털) + Illustrator/InDesign(인쇄물)
4. 캠페인: Figma(디자인/협업) -> 인쇄 도구(오프라인)
5. Figma Draw가 Illustrator 필요성을 점점 줄여가는 중
