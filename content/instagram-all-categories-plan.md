## 2. 행사 — "현장감 하이라이트형"

### 톤
행사는 **에너지와 규모감**이 핵심. 사진의 힘이 세기 때문에 현장 사진 중심으로 가되, "기획자 시점"을 더한다.

### 타이틀 카피
```
메인: 브랜드 행사,
      그냥 하면 아무도 안 온다

서브: 기획부터 운영까지, 사람이 모이는 행사의 비밀
```

### 슬라이드 구성 (프로젝트 3개+ 기준)

| 장 | 내용 | 디자인 |
|---|------|--------|
| 1 | 타이틀 + 행사 현장 전경 사진 | 메인카피 + 서브카피 |
| 2 | 프로젝트A — 대표 현장 사진 | 메인: 행사명, 서브: 규모·장소·참석자 수 |
| 3 | 프로젝트A — 하이라이트 컷 (프로그램/무대/체험) | 사진 중심, 캡션 최소 |
| 4 | 프로젝트B — 대표 현장 사진 | 메인: 행사명, 서브: 규모·장소 |
| 5 | 프로젝트B — 하이라이트 컷 | 사진 중심 |
| 6 | 프로젝트C — 대표 사진 1장 | 행사명 + 규모 |
| 7 | 수치 요약 (or 고정댓글 대체 가능) | "총 OO건 행사 / OO만명 참석 / 만족도 OO%" |
| 8 | CTA | "기억에 남는 행사, 함께 만들어요" |

### 프로젝트 1~2개만 있을 때
- 1개 프로젝트에 3~4장 할당 (다양한 앵글: 전경/무대/참석자/비하인드)
- 또는 Option B(케이스스터디형)로 전환

### 고정 댓글
```
행사는 이렇게 합니다.

1. 컨셉 & 프로그램 기획
2. 장소 섭외 & 협찬 연결
3. 현장 운영 (스태프·진행·안전)
4. 참석자 관리 & 후속 리포팅

기획부터 운영까지 원스톱으로 진행합니다.
```

### 캡션 내 수치 예시
```
📊 행사 OO건+ | 누적 참석 OO만명+ | 평균 만족도 OO%
```

## 3. 메타광고 운영 — "데이터 전문가형"

### 톤
메타광고는 **숫자가 콘텐츠**. 감성적 사진이 아니라 그래프·수치·비교가 메인. 전문성과 신뢰감 중심.

### 타이틀 카피
```
메인: 광고비는 쓰는데
      성과가 안 나온다면

서브: ROAS를 바꾼 Meta 광고 운영 사례
```

### 슬라이드 구성

| 장 | 내용 | 디자인 |
|---|------|--------|
| 1 | 타이틀 — 광고 관리자 대시보드 블러 배경 | 공감형 카피 |
| 2 | 프로젝트A — Before 수치 (ROAS/CPA) | 다크 배경 + 빨간 숫자 (문제) |
| 3 | 프로젝트A — After 수치 | 다크 배경 + 초록 숫자 (개선) |
| 4 | 프로젝트B — Before/After 한 장 요약 | 좌우 비교 레이아웃 |
| 5 | 프로젝트C — Before/After 한 장 요약 | 좌우 비교 레이아웃 |
| 6 | 핵심 인사이트 1개 | "OO 업종은 리타겟팅보다 유사타겟이 OO% 효율적" |
| 7 | 전체 실적 요약 | "총 광고 운용액 OO억 / 평균 ROAS OO% / OO개 브랜드" |
| 8 | CTA | "광고 성과, 숫자로 증명합니다" |

### 프로젝트 1~2개만 있을 때
- 1개 프로젝트를 월별 변화로 풀기 (1개월→2개월→3개월 성과 추이)
- 소재 A/B 테스트 비교 장 추가

### 고정 댓글
```
Meta 광고, 이렇게 운영합니다.

1. 현황 진단 — 기존 캠페인 분석·퍼널 점검
2. 캠페인 구조 설계 — 퍼널별 캠페인 세팅
3. 소재 전략 — A/B 테스트 기반 최적화
4. 성과 리포팅 — 주간/월간 데이터 리포트

숫자로 증명하는 퍼포먼스 마케팅.
```

### 디자인 특이사항
- **다크 배경 권장** (다른 카테고리와 차별화, 전문가 느낌)
- 수치는 크게 (48pt+), 설명은 작게
- 그래프/차트 시각화 활용

## 4. CRM — "교육 인사이트형"

### 톤
CRM은 눈에 보이는 결과물(사진)이 없어서, **개념 설명 + 시스템 구조 시각화**로 풀어야 한다. 교육 콘텐츠처럼 구성해서 저장 유도.

### 타이틀 카피
```
메인: 신규 고객만 쫓다가
      기존 고객을 놓치고 있진 않나요?

서브: 재구매율을 바꾼 CRM 자동화 사례
```

### 슬라이드 구성

| 장 | 내용 | 디자인 |
|---|------|--------|
| 1 | 타이틀 — 공감형 질문 | 미니멀 배경 + 텍스트 중심 |
| 2 | 문제 제기 — "대부분의 브랜드가 빠지는 함정" | 신규 유입에만 집중하는 구조 시각화 |
| 3 | 해결 — CRM 자동화 플로우 (간단 다이어그램) | 세그먼트 → 메시지 → 전환 흐름 |
| 4 | 프로젝트A — Before/After 수치 | 재구매율 OO% → OO% |
| 5 | 프로젝트B — Before/After 수치 | LTV or 이탈률 변화 |
| 6 | 핵심 팁 — "CRM 시작할 때 이것부터" | 저장 유도 콘텐츠 |
| 7 | 실적 요약 | "재구매율 평균 OO% 개선 / OO개 브랜드" |
| 8 | CTA | "고객이 다시 오는 구조, 함께 만들어요" |

### 프로젝트 1~2개만 있을 때
- 교육 비중을 높임 (2~3장: 문제→해결→팁, 4~5장: 사례)
- "CRM 자동화 시나리오 예시"를 시각화하면 저장률 높음

### 고정 댓글
```
CRM, 이렇게 구축합니다.

1. 고객 데이터 분석 — RFM 기반 세그먼트
2. 시나리오 설계 — 구매 여정별 자동화 플로우
3. 채널 세팅 — 카카오·이메일·문자 연동
4. 성과 추적 — 재구매율·LTV·이탈률 모니터링

고객이 다시 찾는 구조를 만듭니다.
```

### 디자인 특이사항
- 텍스트 + 다이어그램 중심 (사진 소재가 적으므로)
- 플로우차트, 세그먼트 표 등 인포그래픽 활용
- 밝은 배경, 깔끔한 톤

## 5. 체험단 — "변화 증명형"

### 톤
체험단은 **Before/After가 가장 극적**으로 보이는 카테고리. 검색 결과 캡처가 곧 증거.

### 타이틀 카피
```
메인: 리뷰 0개에서
      검색 1페이지까지

서브: 체험단으로 검색 지형을 바꾼 사례
```

### 슬라이드 구성

| 장 | 내용 | 디자인 |
|---|------|--------|
| 1 | 타이틀 | 네이버 검색 결과 블러 배경 |
| 2 | Before — 검색 결과 캡처 (리뷰 없음/하위) | 빨간 강조 or 회색 톤 |
| 3 | 우리의 접근 — 체험단 운영 요약 | "OO명 모집 / OO일 운영 / 가이드라인 제공" |
| 4 | 실제 리뷰 콘텐츠 모음 (블로그/인스타 캡처 3~4개 그리드) | 리뷰 품질 보여주기 |
| 5 | After — 검색 결과 캡처 (1페이지 점령) | 초록 강조 |
| 6 | 프로젝트B — Before→After 한 장 요약 | 좌우 비교 |
| 7 | 성과 요약 | "리뷰 OO건 생성 / 검색 OO위 → O위 / 전환 OO% 상승" |
| 8 | CTA | "검색에서 찾아지는 브랜드, 함께 만들어요" |

### 프로젝트 1~2개만 있을 때
- 1개 프로젝트를 깊게 (Before 2장 → 과정 2장 → After 2장)
- 리뷰 콘텐츠 그리드를 늘려서 양적 임팩트

### 고정 댓글
```
체험단, 이렇게 운영합니다.

1. 목표 설정 — 검색 키워드·채널 선정
2. 체험단 모집 — 품질 중심 선발 (팔로워 수 < 콘텐츠 퀄리티)
3. 가이드라인 제공 — 자연스러운 리뷰 유도
4. 콘텐츠 모니터링 — 품질 관리·광고 표기 확인

리뷰가 쌓이면, 검색이 바뀝니다.
```

## 6. 인플루언서 씨딩 — "성과 비교형"

### 톤
"대형 인플루언서 1명 vs 마이크로 10명" 같은 **비교 구도**가 주목을 끈다. 비용 효율을 숫자로 보여주는 톤.

### 타이틀 카피
```
메인: 인플루언서 마케팅,
      돈만 쓰고 끝나지 않으려면

서브: 비용 대비 성과를 만든 씨딩 사례
```

### 슬라이드 구성

| 장 | 내용 | 디자인 |
|---|------|--------|
| 1 | 타이틀 | 인플루언서 콘텐츠 콜라주 배경 |
| 2 | 비교 — 대형 1명 vs 마이크로 10명 (비용/도달/참여율) | 표 or 인포그래픽 |
| 3 | 프로젝트A — 협업 인플루언서 콘텐츠 그리드 | 실제 포스트 캡처 4~6개 |
| 4 | 프로젝트A — 성과 수치 | 도달 OO만 / 참여율 OO% / CPE OO원 |
| 5 | 프로젝트B — 콘텐츠 + 성과 한 장 | 사진 + 수치 오버레이 |
| 6 | 핵심 팁 — "인플루언서 고를 때 이것만 보세요" | 저장 유도 |
| 7 | 실적 요약 | "OO건 캠페인 / 누적 도달 OO만 / 평균 CPE OO원" |
| 8 | CTA | "효율 높은 씨딩, 설계부터 함께" |

### 프로젝트 1~2개만 있을 때
- 교육 비중 높임: "인플루언서 선별 기준 3가지" 같은 팁 장 추가
- 2장의 비교 인포그래픽이 핵심 (이것만으로도 콘텐츠 성립)

### 고정 댓글
```
인플루언서 씨딩, 이렇게 합니다.

1. 전략 설계 — 목표·예산·타겟 맞춤 플래닝
2. 인플루언서 리서치 — 참여율·콘텐츠 퀄리티 기반 선별
3. 브리핑 & 관리 — 가이드라인 제공·일정 관리
4. 성과 분석 — 도달·참여·전환 리포팅 + 2차 활용

제대로 된 씨딩은 광고보다 강합니다.
```

## 7. 공구 (공동구매) — "타임라인 속도감형"

### 톤
공구는 **속도와 매출 숫자**가 핵심. 짧은 시간에 폭발적 성과를 낸 임팩트를 속도감 있게 전달.

### 타이틀 카피
```
메인: 48시간 만에
      3,000만원

서브: 인스타 공구로 매출을 만든 방법
```

### 슬라이드 구성

| 장 | 내용 | 디자인 |
|---|------|--------|
| 1 | 타이틀 — 큰 숫자 + 시간 | 다크 배경, 숫자 강조 |
| 2 | 프로젝트A — 제품 + 셀러 콘텐츠 캡처 | 셀러 포스트/스토리 |
| 3 | 프로젝트A — 실시간 반응 캡처 (댓글 폭주, DM) | 스크린샷 콜라주 |
| 4 | 프로젝트A — 매출 타임라인 | "오픈 1시간: OO만원 / 24시간: OO만원 / 48시간: 매진" |
| 5 | 프로젝트B — 제품 + 성과 한 장 | 사진 + 매출 수치 |
| 6 | 프로젝트C — 제품 + 성과 한 장 | 사진 + 매출 수치 |
| 7 | 전체 실적 | "총 공구 OO건 / 누적 매출 OO억 / 평균 매진 시간 OO시간" |
| 8 | CTA | "매출이 되는 공구, 설계부터 함께" |

### 프로젝트 1~2개만 있을 때
- 1개 프로젝트의 타임라인을 더 세밀하게 (오픈 전→오픈 직후→중간→마감)
- 실시간 반응 캡처를 2~3장으로 늘려 라이브 현장감

### 고정 댓글
```
공구, 이렇게 진행합니다.

1. 제품 분석 — 공구 적합성·가격 설계
2. 셀러 매칭 — 타겟 맞춤 셀러 선정·협업 구조
3. 콘텐츠 세팅 — 오픈 예고·본 콘텐츠·리마인드
4. 운영 & 정산 — 주문 관리·CS·정산

기획부터 정산까지 원스톱으로.
```

### 디자인 특이사항
- 숫자를 매우 크게 (타이틀 장, 타임라인 장)
- 속도감: 화살표, 타임라인 그래픽 활용
- 다크 or 강렬한 컬러 배경 (에너지 표현)

## 8. 브랜드 리뉴얼 — "트랜지션형"

### 톤
리뉴얼은 **Before→After 비주얼이 전부**. 변화의 폭이 클수록 임팩트. 전략적 사고 과정을 살짝 보여주면 전문성도 전달.

### 타이틀 카피
```
메인: 같은 브랜드,
      완전히 다른 인상

서브: 전략부터 디자인까지, 브랜드 리뉴얼 사례
```

### 슬라이드 구성

| 장 | 내용 | 디자인 |
|---|------|--------|
| 1 | 타이틀 — Before/After 패키지 나란히 | 좌: 이전(회색 톤), 우: 이후(컬러) |
| 2 | 프로젝트A — Before (올드 패키지/로고) | "BEFORE" 라벨, 약간 어둡게 |
| 3 | 프로젝트A — After (새 패키지/로고) | "AFTER" 라벨, 밝고 선명하게 |
| 4 | 프로젝트A — 변화 포인트 요약 | "타겟 재정의: 30대→20대 / 톤앤매너: 클래식→모던" |
| 5 | 프로젝트B — Before→After 한 장 | 좌우 분할 비교 |
| 6 | 프로젝트C — Before→After 한 장 | 좌우 분할 비교 |
| 7 | 리뉴얼 후 성과 | "매출 OO% 상승 / SNS 반응 OO% 증가 / 신규 고객 OO% 유입" |
| 8 | CTA | "브랜드의 다음 단계, 함께 설계해요" |

### 프로젝트 1~2개만 있을 때
- 1개 프로젝트의 변화 과정을 세밀하게 (리서치→컨셉→시안→최종)
- 디자인 시안 발전 과정(초안→수정→최종) 공개가 효과적

### 고정 댓글
```
브랜드 리뉴얼, 이렇게 합니다.

1. 브랜드 진단 — 시장 변화·소비자 인사이트 분석
2. 전략 수립 — 포지셔닝·타겟·핵심 가치 재정의
3. 디자인 개발 — 로고·패키지·가이드라인
4. 런칭 & 반응 분석 — 시장 반응·매출 변화 추적

겉모습이 아니라 전략부터 바꿉니다.
```

### 디자인 특이사항
- Before/After 대비를 극대화 (밝기, 색감 차이)
- Before 쪽: 약간 desaturated / After 쪽: 선명하게
- 1장 타이틀에서 바로 비교를 보여주면 스크롤 유도 효과
