# 보고서 생성 파이프라인

유형별 구조화된 보고서를 생성. 숫자 → 해석 → 액션 아이템 원칙.

## 실행 흐름

### 0단계: 유형 판별 + 컨텍스트 로딩

**사용자에게 확인:**
- **클라이언트/브랜드명**
- **보고서 유형**: 주간 / 월간 / 캠페인 결과 / 클라이언트 보고 / 자체 운영
- **기간**: 보고 대상 기간
- **데이터 소스**: 어디서 데이터를 가져올 것인가

**컨텍스트 자동 로딩** (범용 라우팅 룰 적용):
- `clients/{name}/` 폴더에서 기존 보고서, 캠페인 기획서, KPI 정보 참조
- 이전 보고서가 있으면 비교 기준으로 활용

### 1단계: 에이전트 호출 (유형별 분기)

| 유형 | 리드 에이전트 | 서포트 에이전트 |
|------|-------------|---------------|
| 주간 보고서 | 📊 퍼포먼스 분석가 | 📋 프로젝트 매니저 |
| 월간 보고서 | 📊 퍼포먼스 분석가 | — |
| 캠페인 결과 | 📊 퍼포먼스 분석가 | ✍ 콘텐츠 크리에이터 |
| 클라이언트 보고 | 📋 프로젝트 매니저 | 📊 퍼포먼스 분석가 |
| 자체 운영 | 📊 퍼포먼스 분석가 | 💰 세일즈 전략가 |

유형에 따라 해당 에이전트를 **병렬 호출**한다.

### 2단계: 보고서 구조

**모든 보고서 공통 원칙:**
1. **숫자 먼저** — 핵심 KPI를 표로 정리
2. **해석 다음** — 숫자가 의미하는 것
3. **액션 마지막** — 다음에 해야 할 것

**주간 보고서:**
- 이번 주 핵심 수치 (KPI 3-5개, 전주 대비 변화율)
- 완료 작업 / 진행 중 작업
- 주요 인사이트
- 다음 주 계획 (우선순위 3개)
- 이슈/리스크

**월간 보고서:**
- Executive Summary (핵심 3줄)
- KPI Dashboard (목표 대비 실적, 전월 대비)
- 채널별 성과 / 콘텐츠 성과 (Top/Bottom)
- 인사이트 & 분석
- 다음 달 제안

**캠페인 결과:**
- 캠페인 개요 (목적, 기간, 예산, 타겟)
- 성과 요약 (목표 대비 달성률)
- 크리에이티브 성과 (어떤 시안이 효과적이었나)
- 학습 사항 + 다음 캠페인 제안

**클라이언트 보고:**
- 프로젝트 현황 / 수행 업무 / 주요 성과
- 다음 기간 계획 / 논의 필요 사항

**자체 운영:**
- 매출 현황 (MRR, 클라이언트 수)
- 클라이언트별 수익성
- 세일즈 파이프라인 / 운영 효율

### 3단계: 포맷 결정 + 최종 산출물

- 데이터가 부족하면 → "어떤 데이터를 수집해야 하는지" 가이드 제공
- 실적 미달 시 → 솔직하게 보고 + 개선안 제시
- 표와 비교를 적극 활용

## 산출물 저장

```
clients/{client-name}/reports/
└── YYYY-MM-DD-{report-type}.md
```
