# 제안서 생성 파이프라인

5개 에이전트가 각 파트를 담당하여 클라이언트 맞춤 제안서를 생성하는 스킬.

## 실행 흐름

### 0단계: 브리핑 수집 + 컨텍스트 자동 로딩

**사용자에게 확인:**
- **클라이언트/회사명**
- **업종/분야**
- **핵심 과제**: 클라이언트가 해결하고 싶은 것
- **예상 예산 범위** (없으면 규모감이라도)
- **특별 요청**: 제안 범위, 필수 포함 서비스 등

**컨텍스트 자동 로딩** (범용 라우팅 룰 적용):
1. `clients/{name}/context/` 또는 `brief.md` / `strategy.md` / `research.md` 참조
2. `new-biz-prep.md`, `post-meeting-wrap.md` 등 상담 기록이 있으면 함께 참조
3. 아무것도 없으면 핵심 정보를 사용자에게 직접 질문

### Part 1: 현황 진단 (🔍 시장 조사관 + 📊 퍼포먼스 분석가)

`market-researcher` + `performance-analyst` 에이전트 **병렬 호출**:
- **시장 환경**: 시장 규모, 성장률, 주요 트렌드
- **경쟁 분석**: 주요 경쟁사 3-5개, 포지셔닝 맵
- **클라이언트 현황**: 현재 마케팅 성과, 강점/약점
- **핵심 과제**: 해결해야 할 top 3 과제

### Part 2: 전략 방향 (✦ 브랜드 전략가)

`brand-strategist` 에이전트 호출 (Part 1 결과 전달):
- **포지셔닝 전략**: 시장에서의 고유한 위치
- **타겟 정의**: 핵심 고객 페르소나 (1-2개)
- **핵심 메시지**: 브랜드 프로미스, 태그라인 제안
- **차별화 포인트**: 경쟁사 대비 유의미한 차이

### Part 3: 실행 계획 (✍ 콘텐츠 크리에이터 + 📋 프로젝트 매니저)

`content-creator` + `project-manager` 에이전트 **병렬 호출** (Part 1-2 결과 전달):
- **채널 전략**: 어떤 채널에 어떤 콘텐츠를
- **콘텐츠 계획**: 월간 콘텐츠 캘린더 예시
- **실행 로드맵**: 주간/월간 마일스톤
- **산출물 목록**: 구체적인 딜리버리 항목

### Part 4: 성과 예측 (📊 퍼포먼스 분석가)

`performance-analyst` 에이전트 호출:
- **KPI 설정**: 단계별 핵심 지표 (인지→관심→전환→수익)
- **목표 수치**: 3개월/6개월 예상 성과
- **측정 방법**: 추적 및 보고 방법
- **벤치마크**: 업계 평균 대비 기대 수준

### Part 5: 투자 안내 (💰 세일즈 전략가)

`sales-strategist` 에이전트 호출:
- **패키지 옵션**: 3-Tier (Essential / Professional / Premium)
  - 각 패키지별 포함 서비스, 가격, 기간 및 조건
- **ROI 분석**: 투자 대비 예상 수익
- **추천 패키지**: 클라이언트 상황에 맞는 추천 + 이유
- **다음 단계**: 계약 프로세스, 시작 일정

### 규제 검수 (해당 시)

식품/건기식/화장품/의료 업종 클라이언트인 경우:
- `food-regulation-advisor` 호출 → 제안서 내 표현 법적 리스크 점검

## 에이전트 호출 방식

```
0단계: 컨텍스트 로딩
  ↓
[병렬] Part 1: 🔍 시장 조사관 + 📊 퍼포먼스 분석가
  ↓
Part 2: ✦ 브랜드 전략가
  ↓
[병렬] Part 3: ✍ 콘텐츠 크리에이터 + 📋 프로젝트 매니저
  ↓
Part 4: 📊 퍼포먼스 분석가
  ↓
Part 5: 💰 세일즈 전략가
  ↓
(해당 시) ⚖️ 규제 검수
```

## 산출물 저장

```
clients/{client-name}/proposals/
└── YYYY-MM-DD-{proposal-name}.md
```

## 작성 원칙

- 전문적이면서도 읽기 쉬운 톤
- 모든 숫자에 근거 첨부, 추정치는 "추정" 표시
- ROI 예측 필수 포함
- 상담 기록이 있으면 클라이언트가 직접 언급한 과제/니즈를 제안서에 반영
