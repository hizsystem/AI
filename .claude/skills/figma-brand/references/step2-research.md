# Step 2: 웹 리서치 + 브랜드 분석

Step 1에서 수집한 자료를 기반으로 브랜드를 분석하고 brand-analysis.md를 생성한다.

## 리서치 순서

### 2-1. 기본 정보 수집
```
WebSearch: "{브랜드명} 브랜드"
WebSearch: "{브랜드명} instagram"
```
- 회사명, 대표, 설립연도, 업종
- 핵심 슬로건/메시지
- 주요 판매 채널

### 2-2. 웹사이트 분석
```
WebFetch: {website_url}
→ 추출 항목: 메인 컬러, 톤/무드, 제품 카테고리, 네이밍 패턴
```

### 2-3. 로고 컬러 추출
첨부된 로고 이미지에서:
- Primary 컬러 (로고의 주요 색상)
- 로고 스타일 (텍스트형/심볼형/조합형)
- 대소문자 사용 패턴

### 2-4. 인스타그램 분석 (스크린샷이 있을 때)
스크린샷에서 시각적으로 분석:
- 팔로워 수, 게시물 수
- 피드 전체 컬러 톤
- 반복 등장하는 소품/배경
- 촬영 스타일 (플랫레이/라이프스타일/프로덕트샷)
- 콘텐츠 유형 비율

### 2-5. 컬러 시스템 설계
수집된 정보로 6컬러 팔레트를 구성한다:

| 슬롯 | 결정 방법 |
|------|---------|
| Primary | 로고에서 추출 |
| Secondary 1 | 웹사이트/인스타에서 추출, 또는 Primary의 보색/유사색 |
| Secondary 2 | 제품/패키지에서 추출, 또는 무드 기반 생성 |
| Secondary 3 | 배경용 밝은 톤 (Primary를 10% opacity로) |
| Secondary 4 | 보조 그레이 |
| Neutral | 블랙 계열 (#1A1A1A) |

각 컬러에 HEX, RGB, CMYK 값을 포함한다.
CMYK는 HEX에서 근사 변환한다 (정밀 CMYK는 인쇄 시 재확인 필요 명시).

### 2-6. 폰트 추천
브랜드 무드에 맞는 폰트 페어링을 추천한다:
- 영문: 로고 폰트 or 무드에 맞는 Google Fonts
- 한글: Pretendard (기본) — Figma에서는 Inter로 대체 렌더링됨을 명시
