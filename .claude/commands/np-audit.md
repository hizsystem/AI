# 네이버플레이스 100점 진단 (NP Audit)

네이버플레이스 URL을 기반으로 13개 항목 100점 만점 자동 채점 + 수동 질문을 수행하고,
**진단 리포트 + 코칭 태스크 + 추천 패키지** 3종 산출물을 생성합니다.

사용자 입력: $ARGUMENTS

---

## Phase 1: URL 파싱 + 데이터 크롤링

### 1-1. URL 파싱

$ARGUMENTS에서 네이버플레이스 URL을 추출한다.

- 지원 형식: `m.place.naver.com/restaurant/{id}/home`, `naver.me/...`, `place.naver.com/...`
- URL에서 place ID를 추출한다
- URL이 없으면 사용자에게 요청: "네이버플레이스 URL을 입력해주세요 (예: https://m.place.naver.com/restaurant/12345678/home)"
- 모드 판별: $ARGUMENTS에 "full"이 명시적으로 있으면 full 모드, 없으면 기본 **lite** 모드

### 1-2. WebFetch 크롤링

WebFetch로 네이버플레이스 페이지를 크롤링한다. 다음 정보를 추출:

| 항목 | 추출 방법 |
|------|----------|
| **매장명** | 페이지 타이틀 / 매장명 영역 |
| **카테고리** | 업종 표시 영역 |
| **주소** | 주소 영역 |
| **리뷰 수** | 방문자 리뷰 수 (숫자만 추출) |
| **AI 브리핑** | AI 브리핑 영역 (있으면 텍스트, 없으면 "OFF") |
| **메뉴 리스트** | 메뉴명 + 가격 + 사진 유무 |
| **소개문** | 매장 소개 텍스트 (글자 수 측정) |
| **영업시간** | 영업시간 정보 유무 + 내용 |
| **편의시설** | 편의시설/부가정보 유무 + 내용 |
| **찾아오는 길** | 길찾기 안내 유무 |
| **키워드** | 소스코드 내 "keywordList" 또는 대표키워드 영역 |

크롤링 실패 시: "페이지를 불러올 수 없습니다. URL을 확인해주세요." 메시지 출력 후 중단.

## Phase 2: 자동 채점 (~45점)

크롤링 데이터를 기반으로 아래 항목을 자동 채점한다.

### S1. 리뷰 (16/20점 자동)

| 리뷰 수 | 점수 |
|---------|------|
| 0개 | 0점 |
| ~100개 | 4점 |
| ~300개 | 8점 |
| ~500개 | 12점 |
| 1,000개+ | 16점 |

### A1. 메뉴 사진 (2/8점 자동)

| 상태 | 점수 |
|------|------|
| 전체 메뉴에 사진 있음 | 2점 |
| 일부만 사진 있음 | 1점 |
| 대부분 사진 없음 | 0점 |

### A2. 대표 키워드 (5/8점 자동)

| 상태 | 점수 |
|------|------|
| 키워드 세팅됨 | 3점 |
| 키워드에 지역명 포함 | +2점 |
| 키워드 미세팅 | 0점 |

### B1. 기본 정보 (6/6점 자동)

| 항목 | 점수 |
|------|------|
| 영업시간 기재 | 2점 |
| 편의시설 기재 | 1점 |
| 찾아오는 길 기재 | 1점 |
| 사업자 인증 (추정) | 2점 |

### B2. 소개문 (5/5점 자동)

| 글자 수 | 점수 |
|---------|------|
| 없음 | 0점 |
| ~500자 | 1점 |
| ~1,000자 | 3점 |
| 1,500자+ | 5점 |

### B3. AI 브리핑 & 매장명 (4/4점 자동)

| 항목 | 점수 |
|------|------|
| AI 브리핑 ON | 2점 |
| 매장명에 지역명 포함 | 1점 |
| 카테고리 세분화 최적화 | 1점 |

### 자동 채점 결과 표시

자동 채점 완료 후, 사용자에게 수집된 정보 요약을 표시한다:

```
--- 자동 수집 완료 ---

매장명: {매장명}
카테고리: {카테고리}
주소: {주소}
리뷰 수: {리뷰 수}개
AI 브리핑: {ON/OFF}
메뉴: {총 N개, 사진 있는 메뉴 M개}
소개문: {N자}
키워드: {키워드 리스트 또는 "미확인"}

자동 채점: {X}점 / 45점
```

## Phase 3: 수동 질문 (7개, ~55점)

**반드시 아래 7개 질문을 한 번에 모두 묻는다.** 하나씩 묻지 않는다.

```
아래 7가지를 확인해주세요. 모르는 항목은 "모름"이라고 적어주시면 됩니다.

Q1. 대표 사진 퀄리티 — 클릭하고 싶은 느낌인가요? (상/중/하)
Q2. 영상 소식이 업로드되어 있나요? (Y/N)
Q3. 다음 기능이 활성화되어 있나요?
    - 네이버 예약 (Y/N)
    - 알림받기 쿠폰 (Y/N)
    - 톡톡/스마트콜 (Y/N)
Q4. 플레이스 광고를 집행하고 있나요? (미집행 / 집행중 / A/B테스트 중)
Q5. 영수증 리뷰 유도 엽서/이벤트가 있나요? (Y/N)
Q6. 소식을 발행하고 있나요? (Y/N) / 소식에 이벤트 쿠폰이 연동되어 있나요? (Y/N)
Q7. 아래 외부 채널을 운영 중인가요? (각 Y/N)
    - 인스타그램
    - 당근마켓 비즈프로필
    - 블로그
    - 네이버 커넥트(포스)
```

### 수동 채점 매핑

| 질문 | 항목 | 채점 기준 |
|------|------|----------|
| Q1 | A1 대표 사진 (+3점) | 상=3, 중=2, 하=0 |
| Q2 | A1 영상 (+3점) | Y=3, N=0 |
| Q3 예약 | S3 솔루션 (+5점) | Y=5, N=0 |
| Q3 쿠폰 | S3 솔루션 (+5점) | Y=5, N=0 |
| Q3 톡톡 | S3 솔루션 (+5점) | Y=5, N=0 |
| Q4 | S2 광고 (15점) | 미집행=0, 집행중=10, A/B테스트=15 |
| Q5 | S1 리뷰 유도 (+4점) | Y=4, N=0 |
| Q6 소식 | A3 소식 (+4점) | Y=4, N=0 |
| Q6 이벤트 | A3 이벤트 (+3점) | Y=3, N=0 |
| Q7 인스타 | X1 외부 채널 (+1점) | Y=1, N=0 |
| Q7 당근 | X1 외부 채널 (+1점) | Y=1, N=0 |
| Q7 블로그 | X1 외부 채널 (+1점) | Y=1, N=0 |
| Q7 커넥트 | X2 커넥트 (+2점) | Y=2, N=0 |

"모름" 응답은 0점 처리하되, 리포트에 "미확인" 표시.

### 5-2. 코칭 태스크 자동 생성

판정이 "즉시 개선" 또는 "개선 필요"인 항목을 아래 매핑표에 따라 자동으로 태스크로 변환한다.

| 항목 조건 | 태스크 | 주차 | 담당 |
|----------|--------|------|------|
| S1 리뷰 유도 없음 (Q5=N) | 영수증 리뷰 가이드 엽서 제작 | week1 | 우리 |
| S2 광고 미집행 (Q4=미집행) | 플레이스 광고 세팅 | week1 | 우리 |
| S3 예약 OFF (Q3 예약=N) | 네이버 예약 활성화 | week1 | 사장님 |
| S3 쿠폰 OFF (Q3 쿠폰=N) | 알림받기 쿠폰 세팅 | week1 | 사장님 |
| S3 톡톡 OFF (Q3 톡톡=N) | 톡톡+스마트콜 활성화 | week1 | 사장님 |
| A1 사진 미흡 (Q1=하) | 대표 사진 재촬영 | week2 | 사장님 |
| A1 영상 없음 (Q2=N) | 5~10초 영상 촬영+소식 업로드 | week2 | 사장님 |
| A2 키워드 미세팅 | 대표 키워드 5개 세팅 | week1 | 우리 |
| A2 키워드 부적합 (지역명 없음 or 유입 키워드 불일치) | 대표 키워드 교체 (에드로그 기반) | week2 | 우리 | A2 |
| A3 소식 없음 (Q6 소식=N) | 이벤트 소식 발행+쿠폰 연동 | week2 | 사장님 |

| B3 매장명 지역명 없음 | 매장명에 지역명 추가 | week1 | 사장님 |
| X1 외부 채널 없음 (Q7 전부 N) | 인스타/당근 비즈프로필 세팅 | week3 | 사장님 |
| X2 커넥트 미등록 (Q7 커넥트=N) | 네이버 커넥트 전환 검토 | week3 | 사장님 |

### 5-3. 패키지 추천

### 전송 방법

GAS 웹앱은 302 리다이렉트 후 응답을 반환한다. 아래 **2단계** 방식을 사용:

1단계: POST로 GAS URL에 전송 → 302 리다이렉트 URL 획득
2단계: 리다이렉트 URL에 GET 요청 → JSON 응답 확인

```bash
# 1단계: POST → 리다이렉트 URL 획득
REDIRECT_URL=$(curl -s -o /dev/null -w "%{redirect_url}" -X POST "https://script.google.com/macros/s/AKfycbxwyDc5kIu3t5YgPEJFMX6PfLmelKgNKEr269Kcnyy64drOvVcPM3A5UBle_lluCVIh/exec" \
  -H "Content-Type: application/json" \
  -d '{위 JSON}')

# 2단계: 리다이렉트 URL에서 결과 확인
RESULT=$(curl -sL "$REDIRECT_URL")
```

- `{"status":"ok","tabs_updated":4}` → "시트 전송 완료" 표시
- 그 외 → "시트 전송 실패 — 아래 JSON을 수동으로 입력해주세요" + JSON 출력

## Phase 7: 클라이언트용 HTML 진단서 생성

`clients/{store-slug}/np-audit-report.html`로 저장한다.

사장님에게 공유하는 깔끔한 1페이지 웹 리포트. 아래 구조의 HTML을 생성한다.
**참고 템플릿**: `clients/mirye-gukbap/np-audit-report.html` — 이 파일의 디자인 구조를 그대로 따른다.

### HTML 구성 섹션

1. **Header**: 매장명 + 날짜 + 상권 + 업종. 모드(lite/full) 배지
2. **Score Hero**: 원형 점수 (등급별 색상: D=빨강, C=주황, B=노랑, A=초록) + 한줄 코멘트
3. **영역별 바 차트**: S/A/B/X 4개 영역 막대 그래프 (득점/배점, 색상으로 구분)
4. **즉시 개선 Top 3**: 빨간 카드, 순위 + 항목명 + 현재 상태 + 개선 액션 + evidence quote
5. **13개 항목 테이블**: 항목명 | 점수 | 판정 배지 (OK=초록, 주의=노랑, 개선필요=주황, 즉시개선=빨강, 미확인=회색)
6. **잘하고 있는 것**: OK 항목 + 격려 포인트
7. **메뉴명 키워드 제안**: 현재 → 제안 테이블
8. **4주 코칭 플랜**: 주차별 태스크 + 담당 배지 (**"저희"**=파란색, **"직접"**=초록색)
9. **추천 패키지**: 패키지명 + 옵션 목록
10. **Footer**: "브랜드라이즈 | 네이버플레이스 최적화 진단"

### 디자인 규칙

- 최대 너비 640px, 모바일 최적화
- 폰트: -apple-system, 'Pretendard', 'Apple SD Gothic Neo'
- 카드 스타일: 흰 배경 + border-radius 16px + 가벼운 그림자
- 담당자 호칭: 내부 "우리" → 클라이언트용 **"저희"**, 내부 "사장님" → 클라이언트용 **"직접"**
- 이모지 사용 금지

## Full 모드 전용: 키워드 심층 진단 (Phase F1~F3)

`/np-audit full {URL}` 실행 시, Phase 5(3종 산출물) 이후 아래 Phase를 추가 수행한다.
사용자에게 에드로그 스크린샷을 요청하여 진행한다.

### Phase F1: 에드로그 데이터 수집

사용자에게 아래 스크린샷을 요청한다:

```
full 모드 키워드 진단을 위해 에드로그 스크린샷이 필요합니다.

필수:
1. 플레이스 히든 키워드 — 기본 정보 + 키워드 순위표
2. 플레이스 히든 지수 — N1/N2/N3 점수 (핵심 키워드 1~2개)

선택 (있으면 더 정밀 분석):
3. 플레이스 순위 비교분석 — 목표 키워드 상위 업체 비교
4. 플레이스 유입 통계 — 채널별/키워드별 유입수
```

스크린샷이 없으면 lite 진단까지만 완료하고, "에드로그 데이터가 있으면 `/np-keyword {매장명}`으로 키워드 심층 진단을 추가 진행할 수 있습니다"를 안내한다.

### Phase F2: 키워드 4단계 분석

`/np-keyword` 커맨드의 Step 1~4를 그대로 수행한다:

1. **히든 키워드 분류** — 유지 / 핵심 기회 / 기회 / 장기 도전 / 부적합
2. **N1/N2 진단** — 키워드 정합성 vs 축적 데이터 판단
3. **경쟁사 비교** — 상위 업체 대비 병목 식별 (비교분석 데이터 있을 때)
4. **액션 플랜** — 대표키워드 교체 제안 + 키워드별 N1/N2 액션 + 4주 플랜

### Phase F3: 통합 리포트 확장
