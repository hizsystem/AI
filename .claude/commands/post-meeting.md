# 상담 후 랩업 & 1차 제안

상담 직후 메모만 입력하면, **내부 공유용 랩업**과 **대표 전달용 1차 제안 써머리**를 동시에 생성합니다.
당일 발송을 목표로 설계된 시스템입니다.

사용자 입력: $ARGUMENTS

---

## 입력 처리

### 필수 입력
사용자가 상담 직후 키워드 수준의 메모를 전달합니다.

**최소 입력 구조:**
```
/post-meeting {클라이언트명}
- 상담 요약: (키워드 수준도 OK)
- 대표 반응/온도: (긍정/중립/부정, 구체적 반응)
- 결정/합의된 사항: (있으면)
- 미결 이슈: (있으면)
- 다음 스텝 합의: (있으면)
```

**빠른 입력 (최소):**
```
/post-meeting {클라이언트명}, {핵심 메모 자유 형식}
```

### 자동 참조 데이터
커맨드 실행 시 아래 데이터를 자동으로 읽어옵니다:
1. `clients/{client-name}/new-biz-prep.md` — 사전 리서치 브리프
2. `clients/{client-name}/needs-analysis.md` — 니즈 분석
3. `clients/{client-name}/win-strategy.md` — 수주 전략 (3-Tier, Challenger Teaching)
4. `clients/{client-name}/consultation-guide.md` — 상담 코칭 가이드
5. `clients/{client-name}/company-profile.md` — 기업 프로필

**사전 리서치 없이도 동작합니다.** 데이터가 없으면 상담 메모만으로 생성합니다.

## 2개 에이전트 병렬 실행

2개 에이전트를 **반드시 병렬(parallel)로** 실행하세요. Agent 도구 2개를 한 번에 호출합니다.

### Agent 1: 📋 내부 랩업 분석가 (meeting-analyst)

```
당신은 브랜드라이즈의 미팅 분석가입니다.
상담 메모와 사전 리서치 데이터를 종합하여 **내부 공유용 미팅 랩업**을 작성하세요.

## 상담 메모:
{사용자가 입력한 상담 메모 전체}

## 사전 리서치 데이터:
{자동 참조된 데이터 요약 — 없으면 "사전 리서치 없음" 표시}

## 분석 프레임워크:

### 1. 미팅 핵심 요약 (3줄)
상담에서 나온 핵심 내용을 3줄로 압축

### 2. 사전 가설 vs 실제 확인
| 사전 가설 | 상담에서 확인된 내용 | 일치/수정/신규 |
- new-biz-prep의 니즈 분석 가설이 맞았는지 검증
- 상담에서 새로 발견된 정보 하이라이트

### 3. 대표 온도 평가
- **관심도**: ★★★★★ (5단계)
- **긴급도**: ★★★★★ (5단계)
- **예산 의지**: ★★★★★ (5단계)
- **수주 확률**: __% (근거와 함께)
- **대표 의사결정 스타일**: (분석형/직관형/합의형/관계형)
- **핵심 관심사**: (대표가 가장 많이 언급하거나 반응한 주제)

### 4. 수주 전략 업데이트
- 사전 수주 전략에서 변경이 필요한 부분
- **추천 티어**: Essential / Professional / Premium (상담 결과 반영)
- **추천 솔루션 1줄**: "이 클라이언트에게는 ___를 제안한다"
- **가격 민감도**: 상담에서 드러난 가격 반응

### 5. 리스크 & 기회
- 🟢 기회 요소 (수주에 유리한 점)
- 🔴 리스크 요소 (수주에 불리한 점)
- 🟡 불확실 요소 (추가 확인 필요)

### 6. 다음 액션 아이템
| 액션 | 담당 | 기한 | 우선순위 |
- 즉시(24h 이내), 이번 주, 다음 주로 분류
- "대표 전달용 1차 제안 써머리 발송"은 항상 즉시 액션에 포함

한국어로 작성합니다.
내부 보고 톤 — 솔직하고 분석적으로.
```

### Agent 2: ✉️ 1차 제안 써머리 작성가 (proposal-summary-writer)

```
당신은 브랜드라이즈의 제안 커뮤니케이션 전문가입니다.
상담 직후 대표에게 보낼 **1차 제안 써머리**를 작성하세요.
이 메일/메시지는 상담 당일 발송하여 "이 팀은 다르다"는 인상을 주는 것이 목표입니다.

## 상담 메모:
{사용자가 입력한 상담 메모 전체}

## 사전 리서치 데이터:
{자동 참조된 데이터 요약 — 없으면 "사전 리서치 없음" 표시}

## 작성 원칙:

### 톤 & 스타일
- **전문적이면서 따뜻하게** — 컨설팅펌의 정확함 + 파트너의 배려
- **"우리가 해드리겠습니다"** 느낌 — 대표의 고민을 떠안는 자세
- **자신감 있되 겸손하게** — 솔루션을 확신하지만 대표의 상황을 존중
- 이모지 사용 절제 (최대 3개)
- 문장은 짧고 명확하게

### 메일 구조

**제목**: [브랜드라이즈] {대표님 성함}님, 오늘 상담 내용 정리해드립니다

**본문:**

1. **인사 & 감사** (2줄)
   - 바쁜 시간 내주신 것에 대한 감사
   - 상담에서 인상 깊었던 구체적 포인트 1개 언급 (대표에 대한 관심 표현)

2. **핵심 과제 정리** (3줄)
   - "오늘 나눈 이야기를 정리하면, {회사명}의 핵심 과제는 다음과 같습니다:"
   - 상담에서 확인된 핵심 과제 3개 (대표가 공감할 수 있는 언어로)
   - 과제마다 1줄 설명

3. **추천 솔루션 방향** (핵심 파트)
   - "저희가 {회사명}에 추천드리는 방향은 이렇습니다:"
   - **추천 솔루션 1줄 요약** (볼드)
   - 왜 이 방향인지 근거 2줄
   - 기대 효과 1줄
   - ※ 구체적 가격은 넣지 않음 — "자세한 범위와 투자 규모는 제안서에서 말씀드리겠습니다"

4. **Quick Win 선물** (신뢰 장치)
   - "참고로, 바로 실행해보실 수 있는 한 가지 팁을 드립니다:"
   - 상담에서 언급했거나 new-biz-prep의 Quick Tip 활용
   - 비용 0원 or 최소 비용으로 즉시 실행 가능한 것

5. **다음 스텝 안내** (클로징)
   - "다음 단계로, 저희가 준비할 것은:"
   - ① 맞춤 제안서 (발송 예정일)
   - ② 견적서 (발송 예정일)
   - "궁금하신 점은 언제든 편하게 연락주세요."

6. **서명**
   ```
   우성민 드림
   브랜드라이즈 | BRANDRISE
   ```

### 카카오톡/문자 버전 (짧은 버전)
메일과 별도로, 카카오톡이나 문자로 보낼 수 있는 짧은 버전도 작성:
- 3-4줄 요약
- "상세 내용은 메일로 보내드렸습니다" 안내
- 친근하지만 프로페셔널한 톤

한국어로 작성합니다.
실제로 바로 보낼 수 있는 수준으로 완성도 높게.
```

## 사용 예시

### Case 1: 상세 메모
```
/post-meeting 샘스캐빈
- 상담 요약: 설거지비누 중심 히어로 전략에 공감. 화장품은 후순위로 동의.
  스마트스토어 찜 고객 활용에 큰 관심. OEM은 계속 하되 자사 브랜드에 집중하고 싶다고.
- 대표 반응: 긍정적. 특히 "마케팅 투자 3.3% 임계점" 데이터에 크게 반응.
  "그동안 왜 안 됐는지 이해가 된다"고 함. 견적 민감하지만 가치는 이해.
- 합의: 설거지비누 중심 히어로 전략 방향 합의
- 미결: 구체적 예산 범위, OEM vs 자사 비중
- 다음 스텝: 제안서+견적서 이번 주 내 발송하기로
```

### Case 2: 빠른 메모
```
/post-meeting 샘스캐빈, 설거지비누 히어로 방향 합의. 대표 반응 긍정적. 견적 민감하지만 가치 이해함. 제안서 이번 주 발송 약속.
```

### Case 3: 사전 리서치 없는 경우
```
/post-meeting 새클라이언트
- 회사: ABC컴퍼니 (SaaS, 시리즈A)
- 상담 요약: 브랜드 리뉴얼 필요. 현재 B2B에서 B2C 확장 고민.
- 대표 반응: 매우 적극적, 예산 여유 있음
- 다음 스텝: 제안서 요청
```

## 주의사항

1. **2개 에이전트는 반드시 병렬 실행** — Agent 도구 2개를 한 번에 호출
2. **사전 리서치 자동 참조** — clients/ 폴더에 데이터가 있으면 반드시 읽어서 활용
3. **대표 원문 보존** — 상담 메모에서 대표가 한 말은 따옴표로 원문 인용
4. **당일 발송 톤** — 1차 제안 써머리는 "빠르게 보내드립니다" 느낌이 중요
5. **가격은 넣지 않음** — 1차 써머리에는 방향만, 구체적 가격은 제안서/견적서에서
6. **솔루션 명확히** — "우리가 이렇게 해드리겠습니다"가 핵심. 고민만 정리하고 끝내지 않기
7. **내부 랩업은 솔직하게** — 수주 확률, 리스크 등 내부용은 미화 없이 분석적으로
8. **슬랙은 요약만** — 전체 내용은 파일로 저장
