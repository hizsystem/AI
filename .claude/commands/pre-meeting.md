# 사전 미팅 리서치

상담 신청 폼 데이터를 기반으로 미팅 전 자동 리서치를 수행합니다.
3개 에이전트가 병렬로 리서치하여 사전 브리프를 생성하고, 슬랙 DM으로 전송합니다.

사용자 입력: $ARGUMENTS

---

## 입력 처리

사용자 입력에서 다음 정보를 추출하세요:

| 필드 | 설명 | 필수 |
|------|------|------|
| **회사명** | 상담 신청 기업 이름 | O |
| **대표자명** | 상담 대표자 | O |
| **업종/서비스** | 무엇을 하는 회사인지 | O |
| **현재 매출 규모** | 대략적 매출 (0 / ~1억 / ~10억 / ~30억 / 30억+) | - |
| **현재 마케팅 현황** | 어떤 마케팅을 하고 있는지 | - |
| **핵심 고민** | 상담에서 해결하고 싶은 문제 | O |
| **웹사이트/SNS** | 회사 URL, 인스타그램 등 | - |

**입력이 부족한 경우**: 회사명 + 핵심 고민만 있으면 리서치를 시작하세요. 나머지는 웹 리서치로 보완합니다.

---

## 병렬 리서치 실행

3개 에이전트를 **반드시 병렬(parallel)로** 실행하세요. Task 도구를 사용하여 3개를 동시에 호출합니다.

### Agent A: 기업 프로파일링 (Explore agent)

```
다음 기업을 웹 리서치하여 프로필을 작성하세요.

회사명: {회사명}
웹사이트: {웹사이트 (있으면)}
업종: {업종}

조사 항목:
1. 기업 개요 (설립연도, 대표, 팀 규모, 투자 단계)
2. 제품/서비스 (주력 상품, 가격대, 타겟 고객)
3. 온라인 현황 (웹사이트 상태, SNS 채널별 팔로워, 콘텐츠 품질)
4. 최근 뉴스/이벤트 (최근 6개월 내)
5. 경쟁사 (직접 경쟁 2-3개사 이름 + 간단 비교)

WebSearch를 적극 활용하세요. 회사명 + 키워드로 검색합니다.
검색어 예시: "{회사명}", "{회사명} 대표", "{회사명} 서비스", "{회사명} 채용"

JSON이 아닌 한국어 서술형으로 작성하세요.
각 항목별 2-3줄로 간결하게 정리합니다.
모르는 정보는 "확인 필요"로 표시합니다.
```

### Agent B: 시장/업종 분석 (Explore agent)

```
다음 업종의 마케팅 현황을 리서치하세요.

업종: {업종}
기업 규모: {매출 규모}
핵심 고민: {핵심 고민}

조사 항목:
1. 시장 규모 및 성장률 (국내 기준)
2. 업종 내 마케팅 트렌드 (2025-2026)
3. 업종 주요 마케팅 채널 (효과적인 채널 Top 3)
4. 업종 벤치마크 (잘하는 브랜드 2-3개, 무엇을 잘하는지)
5. 업종 특수 고려사항 (규제, 시즌성, 고객 특성)

WebSearch를 적극 활용하세요.
검색어 예시: "{업종} 시장 규모", "{업종} 마케팅 트렌드 2026", "{업종} 브랜드 마케팅 사례"

한국어 서술형으로 2-3줄씩 간결하게 정리합니다.
```

### Agent C: 페인포인트 분석 & 솔루션 매핑 (Explore agent)

```
상담 신청자의 핵심 고민을 분석하고, brandrise 솔루션을 매핑하세요.

회사명: {회사명}
업종: {업종}
매출 규모: {매출 규모}
현재 마케팅: {현재 마케팅 현황}
핵심 고민: {핵심 고민}

분석 항목:
1. 고민 진단: 표면적 고민 vs 근본 원인 구분
2. 유사 케이스: 비슷한 규모/업종에서 흔히 겪는 문제 패턴
3. brandrise 솔루션 매핑:
   - 브랜드 전략 (포지셔닝, 메시징)
   - 콘텐츠 마케팅 (SNS, 블로그, 뉴스레터)
   - 퍼포먼스 마케팅 (메타 광고, 구글 광고)
   - GTM 전략 (론칭, 캠페인)
   중 어떤 서비스가 가장 적합한지, 우선순위를 매기세요.
4. 미팅 시 물어볼 핵심 질문 3개 (진단을 정교하게 할 질문)
5. 제안 방향: 첫 미팅에서 제시할 수 있는 Quick Win 1가지

WebSearch를 활용하여 유사 업종 마케팅 사례를 찾으세요.

한국어 서술형으로 작성하세요.
```

## 결과 종합

3개 에이전트의 결과를 종합하여 **사전 미팅 브리프**를 작성합니다.

### 파일 저장

`clients/{company-name}/pre-meeting-brief.md` 에 저장합니다.
(company-name은 영문 소문자, 하이픈 구분)

### 브리프 구조

```markdown
# 사전 미팅 브리프 — {회사명}

> 생성일: {YYYY-MM-DD}
> 상담 예정일: (알 수 있다면)
> 대표자: {대표자명}

---

## 1. 기업 프로필
{Agent A 결과 요약}

## 2. 시장 환경
{Agent B 결과 요약}

## 3. 핵심 고민 분석
{Agent C 고민 진단 결과}

## 4. 솔루션 매핑
| 우선순위 | 서비스 | 이유 |
|---------|--------|------|
| 1 | {서비스} | {이유} |
| 2 | {서비스} | {이유} |

## 5. 미팅 전략
- **키 질문**: {Agent C의 핵심 질문 3개}
- **Quick Win 제안**: {바로 제시할 수 있는 가치}
- **예상 패키지**: Essential / Professional / Premium 중

## 6. 참고 링크
- {리서치 중 발견한 유용한 URL들}
```

## 사용 예시

### 기본 사용
```
/pre-meeting 주식회사 그린팜, 친환경 농산물 D2C, 매출 5억, 인스타 마케팅 중인데 광고 효과가 없다
```

### 상담 폼 복붙
```
/pre-meeting
회사명: 주식회사 그린팜
대표: 김대표
업종: 친환경 농산물 D2C
매출: 5억
현재 마케팅: 인스타그램 + 블로그
고민: 광고비는 늘리는데 ROAS가 계속 떨어짐, 브랜딩이 필요한 건지 퍼포먼스를 바꿔야 하는 건지 모르겠음
웹사이트: greenfarm.co.kr
```

### 최소 입력
```
/pre-meeting 그린팜, 농산물, 광고 효과 없음
```

## 주의사항

1. **3개 에이전트는 반드시 병렬 실행** — Task 도구 3개를 한 번에 호출
2. **리서치는 WebSearch 위주** — 실시간 정보가 중요
3. **브리프는 스캔 가능하게** — 미팅 5분 전에 훑어볼 수 있도록 짧고 핵심적으로
4. **솔루션 매핑은 brandrise 서비스 기준** — 브랜드 전략 / 콘텐츠 / 퍼포먼스 / GTM
5. **슬랙 DM은 요약만** — 전체 브리프는 파일로 저장
