# 인스타그램 무드보드 AI 워크플로우 시스템

> 작성일: 2026.03.10
> 목적: AI 기반 인스타그램 무드보드 기획 → 서비스 패키지 + 템플릿 판매 상품화
> 첫 적용 대상: 베지어트 (기존 인스타그램 계정 리뉴얼)

---

## 0. 툴 전략: 최소로 시작, 병목에서만 확장

**원칙: 툴이 많을수록 느려진다. 병목이 확인된 지점에만 추가한다.**

### 파일럿 (베지어트): Claude + Figma + 수동 수집

| 역할 | 툴 | 비고 |
|------|---|------|
| 전략/기획/콘텐츠 설계 | **Claude** | 이미 사용 중 |
| 레퍼런스 수집 | **수동** (Pinterest + 인스타 탐색탭) | 시간 측정해둘 것 |
| 컬러 팔레트 | **Coolors.co** (무료) | 30초면 충분 |
| 최종 조립 & 템플릿 | **Figma** | 핵심 산출물 허브 |

### 2번째 프로젝트~: 병목 기반 확장

```
파일럿 회고 후 판단:

"레퍼런스 수집에 2시간 이상 걸렸다"
  → Manus 도입 (웹 브라우징 + 자동 수집)

"무드에 맞는 이미지를 못 찾겠다"
  → Hixfield 도입 (AI 이미지 생성)

"컬러/타이포 조합이 어렵다"
  → Pomelli 도입 (디자인 어시스턴트)

"별로 병목 없었다"
  → 현행 유지. 툴 추가 안 함.
```

## 1. 전체 구조: 3단계 워크플로우

```
[STEP 1] 브랜드 분석 & 채널 컨셉        ← Claude
        ↓
[STEP 2] 콘텐츠 아키텍처 & 시리즈 설계   ← Claude
        ↓
[STEP 3] 비주얼 무드보드 & 레퍼런스 수집  ← 수동 + Coolors
        ↓
[OUTPUT] 피그마 무드보드 + 콘텐츠 템플릿  ← Figma
```

---

## 2. 각 단계별 상세

### STEP 1: 브랜드 분석 & 채널 컨셉 (Claude)

| 작업 | 산출물 |
|------|--------|
| 브랜드 핵심 키워드 추출 | 브랜드 키워드 맵 |
| 타겟 오디언스 페르소나 | 페르소나 카드 (2-3개) |
| 채널 컨셉 스테이트먼트 | 한 문장 컨셉 + 무드 키워드 5개 |
| 경쟁/벤치마크 계정 리스트 | 10-15개 (수동 탐색 또는 추후 Manus) |

**Claude 프롬프트:**
```
[브랜드명]의 인스타그램 채널 컨셉을 잡아줘.
- 브랜드 소개: [간단 설명]
- 타겟: [타겟 설명]
- 기존 톤: [현재 느낌]
- 원하는 방향: [방향성]

아래 형식으로 정리해줘:
1. 채널 컨셉 (한 문장)
2. 무드 키워드 5개
3. 컬러 방향 (메인 2 + 포인트 1)
4. 톤앤매너 가이드 (3줄)
5. DO / DON'T 리스트
```

---

### STEP 2: 콘텐츠 아키텍처 & 시리즈 설계 (Claude)

| 작업 | 산출물 |
|------|--------|
| 콘텐츠 필러(기둥) 설정 | 3-5개 콘텐츠 카테고리 |
| 시리즈 기획 | 시리즈별 콘셉트 시트 |
| 콘텐츠 믹스 비율 | 비율표 (교육:영감:판매:소통) |
| 월간 콘텐츠 캘린더 틀 | 템플릿 캘린더 |

**콘텐츠 아키텍처 프레임워크:**
```
[브랜드 채널 컨셉]
    ├── 필러 1: [카테고리명] (비율: 40%)
    │   ├── 시리즈 A: [시리즈명] - 포맷: 카드뉴스
    │   └── 시리즈 B: [시리즈명] - 포맷: 릴스
    ├── 필러 2: [카테고리명] (비율: 30%)
    │   ├── 시리즈 C: [시리즈명] - 포맷: 싱글이미지
    │   └── 시리즈 D: [시리즈명] - 포맷: 캐러셀
    └── 필러 3: [카테고리명] (비율: 30%)
        ├── 시리즈 E: [시리즈명] - 포맷: 스토리
        └── 시리즈 F: [시리즈명] - 포맷: 릴스
```

---

### STEP 3: 비주얼 무드보드 & 레퍼런스 수집

**파일럿 방식 (수동):**

| 작업 | 방법 | 산출물 |
|------|------|--------|
| 레퍼런스 수집 | Pinterest + 인스타 탐색탭 직접 탐색 | 시리즈별 레퍼런스 5-10개 |
| 컬러 팔레트 | Coolors.co | 팔레트 2-3안 |
| 타이포그래피 | 눈썰미 + Google Fonts | 폰트 페어링 1-2안 |
| 레이아웃 패턴 | Pinterest에서 "instagram carousel layout" 검색 | 레이아웃 레퍼런스 5-10개 |

**핵심: 시리즈별로 레퍼런스를 묶는다**

각 시리즈마다:
- 실제 인스타그램 레퍼런스 5-10개
- 컬러/타이포 방향
- 이렇게 하면 "한 판에 정리"가 시리즈 단위로 구조화됨

**⏱ 파일럿 때 반드시 측정할 것:**
- 레퍼런스 수집에 걸린 시간
- "원하는 느낌"을 찾기까지 탐색한 횟수
- → 이 데이터가 Manus/Hixfield 도입 여부를 결정

---

### 추후 도입 가능: Manus (레퍼런스 수집 자동화)

수동 수집이 병목으로 확인될 경우에만 도입.

**Manus 프롬프트 패턴 (미리 준비):**
```
[브랜드명]의 인스타그램 무드보드를 위한 레퍼런스를 수집해줘.

브랜드 컨셉: [컨셉]
무드 키워드: [키워드 5개]
콘텐츠 시리즈:
- 시리즈 A: [설명]
- 시리즈 B: [설명]
- 시리즈 C: [설명]

각 시리즈별로:
1. 비슷한 톤앤매너의 인스타그램 게시물 5-10개 찾기
2. 각 게시물의 URL, 스크린샷, 왜 레퍼런스로 적합한지 설명
3. 전체적인 비주얼 패턴 분석 (컬러, 레이아웃, 타이포)
```

### 추후 도입 가능: Hixfield (AI 무드 이미지 생성)

"원하는 느낌의 이미지를 못 찾겠다"가 반복될 경우에만 도입.

**활용 패턴:**
```
1. 무드 키워드 → 무드 이미지 생성 (10-20장)
2. 생성된 이미지에서 컬러 추출 → 팔레트 확정
3. 확정된 스타일로 콘텐츠 시안 생성
```

## 3. Figma 템플릿 구조 설계

### 3-1. 무드보드 템플릿 (서비스 산출물 + 판매용)

```
[피그마 무드보드 템플릿 구조]

Page 1: 브랜드 오버뷰
├── 브랜드 키워드 맵
├── 타겟 페르소나 카드
├── 채널 컨셉 스테이트먼트
└── DO / DON'T 가이드

Page 2: 비주얼 무드
├── 무드 이미지 그리드 (3x4 또는 4x5)
├── 컬러 팔레트 (메인 + 서브 + 포인트)
├── 타이포그래피 가이드
└── 사진/일러스트 톤 가이드

Page 3: 콘텐츠 아키텍처
├── 필러 & 시리즈 구조도
├── 콘텐츠 믹스 비율 차트
└── 월간 캘린더 프레임

Page 4~N: 시리즈별 레퍼런스 보드
├── 시리즈 A 레퍼런스
├── 시리즈 B 레퍼런스
├── 시리즈 C 레퍼런스
└── ...
```

### 3-2. 콘텐츠 템플릿 (카드뉴스용, 판매용)

```
[피그마 카드뉴스 템플릿 구조]

Component Set:
├── 커버 슬라이드 (3 Variants: 텍스트형, 이미지형, 하이브리드)
├── 본문 슬라이드 (4 Variants: 텍스트, 인용, 리스트, 비교)
├── CTA 슬라이드 (2 Variants: 팔로우 유도, 링크 유도)
└── 공통 요소
    ├── 로고 마크
    ├── 시리즈 태그
    ├── 페이지 넘버
    └── 컬러/폰트 토큰

→ Design Token만 바꾸면 브랜드별 커스터마이징 가능
→ 이것이 판매 상품의 핵심 가치
```

### 3-3. Figma 핵심 기능 활용

| 기능 | 용도 | 왜 중요한가 |
|------|------|------------|
| **Variables** | 컬러/타이포를 변수로 관리 | 브랜드별 토큰만 교체 → 즉시 커스터마이징 |
| **Components + Variants** | 시리즈별 템플릿 시스템 | 한 번 만들면 무한 재사용 |
| **Auto Layout** | 반복 구조 자동화 | 콘텐츠 추가/삭제 시 자동 정렬 |
| **Unsplash 플러그인** | 무드 이미지 직접 삽입 | 검색 → 드래그 한 번 |

## 4. 상품화 전략: 투 트랙

### 트랙 A: 서비스 패키지 (고가, B2B)

| 패키지 | 포함 내용 | 예상 가격대 |
|--------|-----------|------------|
| Basic | 채널 컨셉 + 무드보드 (Page 1-2) | 50-80만원 |
| Standard | Basic + 콘텐츠 아키텍처 + 시리즈 레퍼런스 (Page 1-N) | 100-150만원 |
| Premium | Standard + 카드뉴스 템플릿 3종 + 1개월 콘텐츠 캘린더 | 200-300만원 |

**작업 시간 목표 (AI 활용 시):**
- Basic: 3-4시간
- Standard: 6-8시간
- Premium: 10-15시간

### 트랙 B: 피그마 템플릿 판매 (저가, B2C)

| 상품 | 내용 | 예상 가격 | 판매 채널 |
|------|------|----------|----------|
| 무드보드 템플릿 | 피그마 무드보드 프레임 (빈 템플릿) | 3-5만원 | 크몽, 탈잉, 자체 |
| 카드뉴스 템플릿 세트 | 시리즈별 10종 세트 | 5-10만원 | 크몽, 탈잉, 자체 |
| 올인원 패키지 | 무드보드 + 카드뉴스 + 사용 가이드 | 15-20만원 | 크몽, 탈잉, 자체 |

**핵심:** 서비스 하면서 만든 것을 범용화해서 판매.
한 번의 작업 → 두 가지 수익.

## 5. 실행 로드맵

### Phase 1: 파일럿 (베지어트) — 1-2주

```
Week 1:
├── Day 1-2: 브랜드 분석 & 채널 컨셉 (Claude)
├── Day 3: 콘텐츠 아키텍처 설계 (Claude)
├── Day 4: 레퍼런스 수집 (수동: Pinterest + 인스타 탐색탭)
│          ⏱ 소요 시간 반드시 기록!
└── Day 5: 컬러 팔레트 (Coolors.co) + 타이포 선정

Week 2:
├── Day 1-2: 피그마 무드보드 조립 + 무드보드 템플릿 v1
├── Day 3-4: 카드뉴스 템플릿 제작
└── Day 5: 클라이언트 전달 + 회고
```

### Phase 1.5: 파일럿 회고 (반나절)

```
필수 체크:
├── ⏱ 어디서 시간이 가장 오래 걸렸나?
├── 🔴 수동 레퍼런스 수집 → 1시간 이상? → Manus 도입 검토
├── 🔴 원하는 이미지 못 찾음? → Hixfield 도입 검토
├── 🔴 컬러/폰트 조합 어려움? → Pomelli 도입 검토
├── 🟢 별 문제 없었다? → 현행 유지
└── 프롬프트 중 재사용 가능한 것 따로 저장
```

### Phase 2: 워크플로우 정착 — 1주

```
├── 프롬프트 라이브러리 정리 (재사용 가능한 것만)
├── 피그마 템플릿 범용화 (브랜드 특정 요소 → Variables로 변수화)
├── 작업 체크리스트 & SOP 문서화
└── (필요 시) 추가 AI 툴 학습 & 테스트
```

### Phase 3: 상품화 & 스케일링 — 2-3주

```
├── 크몽/탈잉 등록용 템플릿 패키지 제작
├── 판매 페이지 카피 작성 (Claude)
├── 포트폴리오용 케이스 스터디 제작 (베지어트 사례)
└── 2번째 클라이언트 수주 & 적용
```
