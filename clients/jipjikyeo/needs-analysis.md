## Step 1. 인테이크 데이터 분석

### 원문 키워드 추출

| 키워드 | 해석 |
|--------|------|
| "신규 유저 유입" | Acquisition — 그로스 단계 진입 욕구 |
| "정확히 어떤 것들을 제공해주는지" | 서비스 가치 전달 실패 자각 |
| "인식 함양" | 단순 노출이 아닌 이해/교육 수준의 커뮤니케이션 필요 |
| "브랜드 마케팅 (BI 개발 등)" | 문제를 "브랜드가 약해서"로 귀인 |
| "현 마케팅 문제에 대한 명확한 진단" | 스스로 뭐가 문제인지 모르겠다는 솔직한 고백 |

### 감정 톤 분석

- **혼란** — 여러 기능(행복주택+보증금진단+대출조회)을 갖고 있으나 뭘로 알려야 할지 결정 못함
- **조급함** — Seed 투자 후 2년 차, 성장 증명 필요한 시점
- **기대감 낮음** — 기대 성과가 "진단"인 것은 당장 결과보다 방향 설정을 원한다는 뜻
- **메일 미회신** — 아직 확신이 없거나 비교 탐색 중

## Step 2. SPIN 기반 니즈 레이어링

### S (Situation) — 상황

- 2023년 런칭, Seed 2억 투자, TIPS 선정된 초기 프롭테크 스타트업
- 보증금 진단 10조원 돌파 등 트랙션은 있음
- 인스타 139K 팔로워 — 콘텐츠 마케팅 채널은 이미 운영 중
- 월 예산 500만원 이하 — 초기 스타트업 전형적 마케팅 예산
- IT/소프트웨어 팀이라 마케팅/브랜딩 전문 인력 부재 가능성 높음

### P (Problem) — 문제

- 서비스 기능이 여러 개(행복주택 검색 + 보증금 진단 + 등기부 알림 + 대출 조회)인데 **하나의 명확한 메시지로 수렴이 안 됨**
- "집지켜"라는 이름만으로는 뭘 하는 서비스인지 직관적으로 알기 어려움
- 인스타 139K가 있어도 앱 다운로드 전환이 월 2.8K에 그침 → 팔로워가 "재미있는 부동산 콘텐츠 소비자"일 뿐 "집지켜 유저"로 전환되지 않을 가능성
- 직방, 다방, 호갱노노 등 대형 부동산 플랫폼 사이에서 차별 포인트 전달 실패

### I (Implication) — 시사점/확대 영향

- 메시지가 불명확하면 CAC(고객획득비용)가 계속 올라감
- 다음 라운드(Pre-A) 투자 유치 시 "성장 지표"를 보여줘야 하는데, 인지도 없이 퍼포먼스만 돌리면 효율이 나지 않음
- 경쟁사(직방 등)가 유사 기능을 추가하면 차별화 창구가 닫힘
- 팀 내부적으로도 "우리가 뭘 파는 건지" 합의가 안 되면 제품 로드맵도 흔들림

### N (Need-Payoff) — 해결 시 기대 효과

- 서비스 한 줄 정의가 명확해지면 → 모든 마케팅 채널의 메시지가 일관되고 → 전환율이 오름
- "보증금 안전 = 집지켜" 카테고리 포지션 확보 → 오가닉 유입 증가
- 브랜드 스토리가 명확하면 → 투자 IR에서도 서비스 설명이 쉬워짐
- 500만원 예산으로도 "선택과 집중"이 가능해짐

## Step 3. 고민 계층 분석 (Pain Hierarchy)

### L1. 표면 고민 (Surface)
> "신규 유저가 안 들어온다" / "서비스가 뭔지 사람들이 모른다"

- 다운로드 수 정체, 인지도 부족 체감
- 대표가 직접 느끼는 가장 즉각적인 통증

### L2. 전술적 문제 (Tactical)
> "어떤 채널에서 뭘 어떻게 말해야 할지 모르겠다"

- 인스타 139K가 있는데도 전환이 안 되는 것 → 채널 전략이 아니라 **메시지 전략**의 문제
- 행복주택? 보증금 진단? 전세대출? 뭘 앞세워야 할지 결정 못함
- "브랜드 마케팅/BI 개발"을 요청한 것은 이 혼란을 "브랜드가 약해서"로 해석한 것

### L3. 전략적 문제 (Strategic)
> "집지켜가 시장에서 어떤 포지션인지 정의가 안 됐다"

- 행복주택 검색 앱? 보증금 안전 진단 앱? 청년 주거 플랫폼? — 정체성 미확립
- 직방/다방 vs 집지켜의 근본적 차별점이 한 문장으로 안 나옴
- 기능은 계속 추가했는데 "왜 집지켜여야 하는지"는 후순위로 밀린 상태

### L4. 근본 문제 (Root)
> "제품과 시장 사이에 '번역'이 없다"

- 개발팀이 만든 기능과 유저가 느끼는 가치 사이의 간극
- IT 창업자의 전형적 맹점: 좋은 제품을 만들면 알아서 쓸 것이라는 가정
- **Product-Market Fit은 있는데 Product-Message Fit이 없는 상태**

## Step 4. JTBD (Jobs To Be Done) 분석

### 대표(김한성)의 Job

| 구분 | 내용 |
|------|------|
| **Functional Job** | 마케팅 비용 대비 신규 유저 유입을 늘리고 싶다 |
| **Emotional Job** | "우리 서비스가 좋은데 왜 안 알려지지?"라는 답답함을 해소하고 싶다 |
| **Social Job** | 투자자/팀원에게 "성장하고 있다"는 것을 보여주고 싶다 |

### 집지켜 유저의 Job (대표가 이해해야 할 것)

| 구분 | 내용 |
|------|------|
| **Core Job** | 전세 사기 안 당하고 안전한 집을 구하고 싶다 |
| **Related Job** | 복잡한 부동산 정보를 쉽게 이해하고 싶다 |
| **Emotional Job** | 처음 독립하는데 불안하지 않고 싶다 |

### JTBD 시사점

대표는 "서비스가 뭘 하는지 알려야 한다"고 말하지만, 유저 관점에서 Job은 **"전세 사기 불안 해소"**라는 감정적 니즈. 기능 나열(행복주택+보증금진단+대출)이 아니라 **"집 구할 때 겁나지 않게 해주는 앱"**이라는 감정 포지셔닝이 더 효과적일 수 있음.

## Step 5. 유사 사례 패턴

### 패턴 A: "기능은 많은데 뭔지 모르겠는" IT 서비스

- **토스 초기**: 송금 앱으로 시작 → "간편 송금 = 토스"로 단일 메시지 집중 → 이후 기능 확장
- **뱅크샐러드 초기**: 자산관리+가계부+대출비교 → "내 돈 관리 = 뱅크샐러드"로 수렴
- **교훈**: 다기능 앱일수록 **진입점(Entry Point) 하나**를 명확히 잡아야 함

### 패턴 B: "브랜드 마케팅 해주세요"라고 왔지만 실제 니즈가 다른 경우

IT 스타트업이 "브랜드 마케팅/BI 개발"을 요청하는 경우, 실제 니즈는 대부분:
1. 서비스 포지셔닝 정리 (70%) — 가장 빈번
2. 랜딩페이지/앱스토어 전환율 개선 (15%)
3. 진짜 로고/BI 리뉴얼 (10%)
4. 투자 IR용 브랜드 스토리 (5%)

집지켜의 경우 **1번(포지셔닝 정리)**이 핵심이고, BI 개발은 그 이후의 선택지.

### 패턴 C: 프롭테크 마케팅의 공통 난제

- 부동산 서비스는 "필요할 때만 쓴다" → 상시 인지도 유지가 어려움
- 전세사기 이슈 등 사회적 화두와 연동되는 타이밍 마케팅이 효과적
- 인스타그램보다 **검색(네이버/구글)** 채널이 전환에 더 가까움 (집 구할 때 검색부터 함)

## Step 6. 검증 질문 5개

상담 시 아래 질문으로 가설을 검증하고 진짜 니즈를 확인한다.

### Q1. 서비스 정의 (포지셔닝 가설 검증)
> "집지켜를 처음 듣는 사람에게 한 문장으로 설명한다면 어떻게 하세요? 그리고 그 설명을 들은 사람의 반응은 보통 어떤가요?"

**의도**: 대표 본인도 한 문장 정의가 안 될 가능성 높음. 정의가 안 된다면 → 포지셔닝이 1순위 문제 확인.

### Q2. 유입 구조 (채널 효율 진단)
> "지금 신규 유저가 들어오는 경로 Top 3가 어디인가요? 인스타 139K 팔로워 중 실제 앱 다운로드로 이어지는 비율은 어느 정도로 보세요?"

**의도**: 인스타 팔로워 수 vs 실제 전환의 괴리 확인. 채널 문제인지 메시지 문제인지 분리.

### Q3. 핵심 기능 우선순위 (진입점 확인)
> "행복주택 검색, 보증금 안전 진단, 등기부 알림, 대출 조회 — 이 중에서 유저가 가장 먼저 쓰는 기능, 가장 많이 쓰는 기능, 가장 만족도 높은 기능이 각각 뭔가요?"

**의도**: 마케팅 메시지의 리드 기능을 데이터 기반으로 특정. "보증금 진단 10조"가 핵심이라면 이걸 전면에 세워야 함.

### Q4. 경쟁 인식 (차별화 가설)
> "유저들이 집지켜 대신 직방이나 다방을 쓰는 이유가 뭐라고 보세요? 반대로 집지켜만 쓰는 유저는 왜 여기 남아있나요?"

**의도**: 대표가 생각하는 차별점과 유저가 실제 느끼는 차별점의 Gap 확인.

### Q5. 기대 수준 & 타임라인 (스코프 조율)
> "6개월 뒤에 집지켜가 어떤 상태이면 성공이라고 느끼실 것 같으세요? 그리고 다음 투자 라운드 일정이 있으신가요?"

**의도**: 단순 브랜딩이 아니라 투자 연계 성장 목표가 있는지 확인. 타임라인에 따라 제안 범위가 달라짐.

## 종합 진단

### 대표가 말한 것 vs 실제 필요한 것

| 대표의 요청 | 실제 니즈 (추정) |
|------------|----------------|
| 브랜드 마케팅 / BI 개발 | 서비스 포지셔닝 정립 + 핵심 메시지 도출 |
| 신규 유저 유입 | 유입 채널별 메시지 최적화 + 전환 퍼널 설계 |
| 서비스 인식 함양 | "집지켜 = ___" 카테고리 연상 구축 |
| 마케팅 문제 진단 | 포지셔닝 → 메시지 → 채널의 전체 프레임 재설계 |

### 우선순위 추천

1. **서비스 포지셔닝 워크숍** — "집지켜 = 보증금 안전 진단" (또는 다른 핵심 기능) 한 줄 정의
2. **메시지 아키텍처** — 채널별(인스타/앱스토어/검색광고) 메시지 체계 설계
3. **전환 퍼널 진단** — 인스타 팔로워 → 앱 다운로드 → 핵심 기능 사용의 흐름 점검
4. **BI 리뉴얼은 보류** — 포지셔닝이 잡힌 후에 시각적 아이덴티티 정리하는 것이 순서

### 예산 적합성 (500만원/월)

- 풀 브랜딩 리뉴얼은 예산 부족
- 포지셔닝 컨설팅 + 메시지 설계 (1~2개월 프로젝트형)는 적합
- 이후 실행 단계는 인하우스 적용 가이드 형태로 전달하면 예산 내 가능

### 리스크 요인

- 메일 미회신 상태 → 아직 의사결정 전 비교 탐색 중
- "진단만 원한다"는 표현 → 당장 큰 비용 집행 의지 낮을 수 있음
- 기술 창업자 특성상 "마케팅 = 비용"으로 인식할 가능성 → ROI 기반 설득 필요
