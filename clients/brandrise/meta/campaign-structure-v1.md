## 1. 솔직한 평가부터: 50만원으로 100건 상담은 비현실적

결론부터 말씀드리면, **Meta 광고 단독으로 월 100건 상담 신청은 50만원 예산으로 불가능**합니다.

### 현실적 수치 계산

| 지표 | 보수적 추정 | 낙관적 추정 |
|------|-----------|-----------|
| 월 예산 | 500,000원 | 500,000원 |
| 예상 CPM (B2B, 한국) | 15,000원 | 10,000원 |
| 총 노출 | 33,333회 | 50,000회 |
| CTR (클릭률) | 1.0% | 1.5% |
| 총 클릭 | 333회 | 750회 |
| CPC (클릭당 비용) | 1,500원 | 667원 |
| 프로필 방문 전환율 | 40% | 50% |
| 프로필 방문 수 | 133회 | 375회 |
| 링크 클릭 → 폼 도달 | 30% | 40% |
| 폼 도달 수 | 40회 | 150회 |
| 폼 완료율 (상담 신청) | 20% | 30% |
| **예상 상담 신청** | **8건** | **45건** |

**현실적 기대치: Meta 광고로 월 10~30건 상담 신청**

> 100건 달성하려면 오가닉(인스타 콘텐츠, DM, 리퍼럴) + 유료 광고 합산이 필요합니다.
> 광고 예산을 월 150~200만원으로 올리거나, 전환 퍼널을 단축(프로필 경유 없이 바로 폼)해야 100건에 접근 가능합니다.

### 예산 대비 현실적 목표 (권장)

| 항목 | 목표 |
|------|------|
| Meta 광고 상담 신청 | 15~25건/월 |
| 오가닉 상담 신청 | 10~15건/월 |
| 합산 | 25~40건/월 |

## 2. 캠페인 구조

### 전체 구조 트리

```
[캠페인] 브랜드라이즈_팝업스토어_리드젠_2603
├── 목표: 트래픽 (프로필 방문 최적화)
├── 예산 방식: ABO (테스트 단계이므로 광고세트별 예산)
│
├── [광고세트 A] 코어_브랜드마케터
│   ├── 일예산: 8,000원
│   ├── 타겟: 브랜드/마케팅 담당자 (관심사 기반)
│   ├── [광고 1] 캐러셀_비비고x세븐틴
│   ├── [광고 2] 캐러셀_네스카페돌체구스토
│   └── [광고 3] 캐러셀_포트폴리오_베스트3
│
├── [광고세트 B] 코어_리테일브랜드
│   ├── 일예산: 8,000원
│   ├── 타겟: 리테일/F&B/뷰티 브랜드 관련자
│   ├── [광고 1] 캐러셀_비비고x세븐틴
│   ├── [광고 2] 캐러셀_네스카페돌체구스토
│   └── [광고 3] 캐러셀_포트폴리오_베스트3
│
└── [광고세트 C] 리타게팅_참여자
    ├── 일예산: 4,000원 (3월 2주차부터 ON)
    ├── 타겟: 인스타 프로필 방문자 + 게시물 참여자
    ├── [광고 1] 상담CTA_클라이언트후기
    └── [광고 2] 상담CTA_작업과정
```

### 왜 이 구조인가

| 결정 | 이유 |
|------|------|
| ABO (광고세트별 예산) | 50만원 소액 테스트 단계. 세트별 성과를 정확히 비교해야 함 |
| 트래픽 목표 (전환 아님) | 픽셀 미설정 상태 + 전환 데이터 부족. 전환 캠페인은 주당 50건 이상 전환 필요 |
| 광고세트 3개 | 2개 코어 타겟 비교 + 1개 리타게팅. 소액에서 4개 이상은 학습 데이터 부족 |
| 광고세트당 3개 소재 | 알고리즘이 최적 소재를 찾을 수 있는 최소 단위 |

## 7. 픽셀 세팅 가이드

### 왜 필요한가

현재 전환 추적이 안 되면 "누가 광고 보고 상담 신청했는지" 알 수 없습니다. 성과 측정과 향후 전환 캠페인 전환을 위해 필수입니다.

### 세팅 단계 (5단계, 30분 소요)

**1단계: 픽셀 생성**
- Meta 비즈니스 관리자 → 이벤트 관리자 → 데이터 소스 → 픽셀 추가
- 픽셀 이름: "brandrise_popup"

**2단계: 상담 신청 폼 페이지에 픽셀 설치**
- 상담 폼이 호스팅된 플랫폼 확인 (Tally, Google Form, 자체 웹 등)
- Tally/Typeform: 내장 Meta 픽셀 연동 기능 사용
- Google Form: 별도 랜딩 페이지 필요 (Google Form 자체는 픽셀 불가)
- 자체 웹: HTML <head>에 픽셀 기본 코드 삽입

**3단계: 전환 이벤트 설정**
- 표준 이벤트: `Lead` (상담 신청 완료 시)
- 폼 완료 Thank You 페이지 URL에 이벤트 트리거 설정
- 코드 예시:
```html
<!-- 폼 완료 페이지에 추가 -->
<script>
  fbq('track', 'Lead');
</script>
```

**4단계: 도메인 인증**
- 비즈니스 관리자 → 브랜드 안전성 → 도메인 → 도메인 추가 및 인증
- DNS TXT 레코드 또는 HTML 메타태그 방식

**5단계: 테스트**
- Meta Pixel Helper (Chrome 확장) 설치 → 폼 페이지 접속 → 이벤트 발생 확인
- 이벤트 관리자에서 테스트 이벤트 탭으로 실시간 확인

### 픽셀 없이 당장 할 수 있는 것

| 방법 | 설명 |
|------|------|
| UTM 파라미터 | 광고 링크에 `?utm_source=meta&utm_medium=paid&utm_campaign=popup_leadgen` 추가 |
| 폼 히든 필드 | "어디서 오셨나요?" 필드 자동 채우기로 유입 경로 추적 |
| 수동 집계 | 주 1회 상담 신청 수 vs 광고 비용 비교 |

## 8. 운영 로드맵

### Phase 1: 테스트 (3월 14일 ~ 3월 31일, 18일)

| 주차 | 액션 |
