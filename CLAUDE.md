# Meta Ads Team — MarketingOS

Meta(Facebook/Instagram) 광고 전문 워크트리. 캠페인 설계, 소재 제작, 성과 분석, 최적화를 담당한다.

## Meta 광고 전담 팀 (4명)

| 에이전트 | 파일 | 역할 |
|---------|------|------|
| (🎯) Meta 미디어바이어 | `meta-media-buyer` | 캠페인 구조, 타겟, 예산, 입찰 전략 |
| (🎨) Meta 크리에이티브 PD | `meta-creative-pd` | 소재 기획, 카피, A/B 테스트 설계 |
| (📈) Meta 그로스 분석가 | `meta-growth-analyst` | 지표 분석, 퍼널 진단, 스케일링 판단 |
| (📋) Meta 어카운트 매니저 | `meta-account-manager` | 온보딩, 리포팅, 클라이언트 커뮤니케이션 |

## 소재 자동 제작 파이프라인 (Gemini 3 Pro)

`meta-ad-pipeline/` 디렉터리에 위치. 5개 에이전트가 순차 실행.

```
정보수집 → 리서치 → 카피라이팅 → 디자인 → 프롬프팅 → 이미지 생성
```

- **오케스트레이터**: `main.py`
- **AI 백엔드**: `gemini_client.py` (Gemini 3 Pro API)
- **산출물**: JSON 5개 + PNG 13개 (카드뉴스 세트)

## 워크플로우 커맨드

| 커맨드 | 용도 |
|--------|------|
| `/meta-onboard` | 신규 클라이언트 Meta 광고 온보딩 (5단계) |
| `/meta-campaign` | Meta 캠페인 구조 자동 설계 (4단계) |
| `/meta-report` | Meta 성과 보고서 생성 (일간/주간/월간) |
| `/meta-optimize` | Meta 최적화 진단 (퍼널 → 에이전트별 액션) |

## 클라이언트별 프로젝트 구조

```
clients/
└── {client-name}/
    ├── brief.md           # 클라이언트 광고 브리프
    ├── campaign-structure/ # 캠페인 구조 (CBO/ABO, 광고세트, 타겟)
    ├── creatives/         # 소재 (이미지, 카피, A/B 변형)
    ├── reports/           # 성과 보고서 (일간/주간/월간)
    └── optimization/      # 최적화 로그 및 액션 플랜
```

## 핵심 지표 (KPI)

- **효율**: CPA, ROAS, CPM, CPC, CTR
- **퍼널**: 노출 → 클릭 → 랜딩 → 전환 (각 단계 전환율)
- **크리에이티브**: 후크율(3초 시청), 소재별 CTR/CVR 비교
- **학습**: 학습 기간(3~7일) 완료 여부, 최적화 이벤트 달성 수

## 작업 규칙

- 캠페인 구조가 먼저, 소재는 그 다음
- 테스트 예산과 스케일링 예산을 항상 구분한다
- 최소 1,000 노출 이후에 성과를 판단한다
- 소재는 항상 2-3개 시안을 제시한다
- 보고서: 숫자 → 해석 → 액션 아이템 순서
- 레퍼런스 분석 결과를 소재 제작에 반영한다

## 커뮤니케이션 원칙

1. **한국어 기본**: 모든 산출물은 한국어
2. **스타트업 눈높이**: 전문용어는 괄호로 쉬운 설명 병기
3. **데이터 기반**: 주장에는 근거를, 제안에는 예상 수치를
4. **실행 가능성 우선**: 이론보다 바로 집행 가능한 결과물
