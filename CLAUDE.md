# MarketingOS

MarketingOS는 스타트업 대상 마케팅/브랜딩 솔루션 서비스다. 1인 운영 체제이며, 이 에이전트 시스템은 서비스 구축과 클라이언트 프로젝트 수행 모두에 활용한다.

## 비즈니스 개요

- **타겟**: 시드~시리즈A 스타트업 (마케팅 전담 인력 없는 팀)
- **핵심 서비스**: 브랜드 전략, 콘텐츠 마케팅, 퍼포먼스 마케팅, GTM 전략
- **운영 방식**: 1인 + AI 에이전트 팀
- **동시 프로젝트**: 최대 3개

## 에이전트 팀 (6명)

| 에이전트 | 파일 | 역할 |
|---------|------|------|
| (🔍) 시장 조사관 | `market-researcher` | 시장분석, 경쟁사, 트렌드 리서치 |
| (✦) 브랜드 전략가 | `brand-strategist` | 포지셔닝, 메시징, 아이덴티티 |
| (✍) 콘텐츠 크리에이터 | `content-creator` | 카피라이팅, SNS, 블로그, 광고 |
| (💰) 세일즈 전략가 | `sales-strategist` | 퍼널 설계, 가격 전략, 제안서 |
| (📋) 프로젝트 매니저 | `project-manager` | 일정, 산출물, 리소스 관리 |
| (📊) 퍼포먼스 분석가 | `performance-analyst` | KPI, ROI, 대시보드, 리포팅 |

## 워크플로우 커맨드

| 커맨드 | 용도 |
|--------|------|
| `/marketingos` | 도움말 및 사용 가이드 |
| `/new-client` | 신규 클라이언트 온보딩 (6단계) |
| `/proposal` | 제안서 생성 (5파트) |
| `/campaign` | 캠페인 기획 (5단계) |
| `/content` | 콘텐츠 제작 (5단계) |
| `/report` | 보고서 생성 (유형별) |
| `/review` | 멀티에이전트 리뷰 (6명 순차 피드백) |

## 커뮤니케이션 원칙

1. **한국어 기본**: 모든 산출물은 한국어로 작성한다
2. **스타트업 눈높이**: 마케팅 전문용어는 괄호 안에 쉬운 설명을 병기한다
3. **실행 가능성 우선**: 이론보다 바로 쓸 수 있는 결과물을 만든다
4. **데이터 기반**: 주장에는 근거를, 제안에는 예상 수치를 붙인다
5. **80/20 원칙**: 핵심 20%에 집중하여 80% 효과를 낸다

## 클라이언트 프로젝트 구조

```
clients/
└── {client-name}/
    ├── brief.md       # 클라이언트 브리프
    ├── research.md    # 시장 조사 결과
    ├── strategy.md    # 브랜드/마케팅 전략
    ├── proposals/     # 제안서
    ├── content/       # 콘텐츠 산출물
    ├── campaigns/     # 캠페인 기획
    └── reports/       # 보고서
```

## 작업 규칙

- 새 클라이언트는 반드시 `/new-client`로 시작한다
- 모든 제안서에는 ROI 예측을 포함한다
- 콘텐츠는 항상 2-3개 시안을 제시한다
- 보고서에는 숫자 → 해석 → 액션 아이템 순서를 따른다
- 주간 리뷰로 프로젝트 상태를 점검한다
