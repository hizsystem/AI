# MarketingOS

MarketingOS는 스타트업 대상 마케팅/브랜딩 솔루션 서비스다. 1인 운영 체제이며, 이 에이전트 시스템은 서비스 구축과 클라이언트 프로젝트 수행 모두에 활용한다.

## 비즈니스 개요

- **타겟**: 시드~시리즈A 스타트업 (마케팅 전담 인력 없는 팀)
- **핵심 서비스**: 브랜드 전략, 콘텐츠 마케팅, 퍼포먼스 마케팅, GTM 전략
- **운영 방식**: 1인 + AI 에이전트 팀
- **동시 프로젝트**: 최대 3개 (스타트업) + 10개+ (소상공인 코치형)

## 커뮤니케이션 원칙

1. **한국어 기본**: 모든 산출물은 한국어로 작성한다
2. **스타트업 눈높이**: 마케팅 전문용어는 괄호 안에 쉬운 설명을 병기한다
3. **실행 가능성 우선**: 이론보다 바로 쓸 수 있는 결과물을 만든다
4. **데이터 기반**: 주장에는 근거를, 제안에는 예상 수치를 붙인다
5. **80/20 원칙**: 핵심 20%에 집중하여 80% 효과를 낸다

    ├── brief.md       # 클라이언트 브리프
    ├── research.md    # 시장 조사 결과
    ├── strategy.md    # 브랜드/마케팅 전략
    ├── proposals/     # 제안서
    ├── content/       # 콘텐츠 산출물
    ├── campaigns/     # 캠페인 기획
    ├── reports/       # 보고서

## 작업 규칙

- 새 클라이언트는 반드시 `/new-client`로 시작한다
- 모든 제안서에는 ROI 예측을 포함한다
- 콘텐츠는 항상 2-3개 시안을 제시한다
- 보고서에는 숫자 → 해석 → 액션 아이템 순서를 따른다
- 주간 리뷰로 프로젝트 상태를 점검한다

## 범용 에이전트 라우팅 룰

클라이언트 작업 시, 아래 규칙에 따라 에이전트를 자동 호출하고 컨텍스트를 참조한다. 클라이언트별 오버라이드가 있으면 범용 규칙 대신 적용한다.

### 클라이언트 컨텍스트 자동 로딩

클라이언트 관련 작업이 감지되면, 해당 `clients/{name}/` 폴더에서 컨텍스트를 자동으로 읽는다.

| 우선순위 | 조건 | 참조 파일 |
|---------|------|----------|
| 1순위 | `context/` 폴더 존재 | 작업 유형에 맞는 context 파일 선택 참조 |
| 2순위 | `context/` 없음 | `brief.md` → `strategy.md` → `research.md` 순서로 참조 |

- 클라이언트명은 사용자 발화에서 감지하거나, 직접 물어서 확인한다
- 컨텍스트 파일이 하나도 없으면 에이전트 호출 전에 먼저 브리프 수집부터 한다

### 작업 유형 → 에이전트 자동 매핑

스킬(`/command`) 없이 자유 대화로 작업을 요청받았을 때, 아래 규칙으로 에이전트를 자동 호출한다.

| 작업 유형 | 트리거 키워드 | 리드 에이전트 | 서포트 (순차 호출) |
|-----------|-------------|-------------|-------------------|
| 리서치/분석 | 시장, 경쟁사, 트렌드, 조사 | 🔍 시장 조사관 | 📊 퍼포먼스 분석가 (수치 해석) |
| 브랜드/전략 | 포지셔닝, 브랜딩, 메시지, 아이덴티티 | ✦ 브랜드 전략가 | 🔍 시장 조사관 (선행 리서치) |
| 콘텐츠 제작 | 콘텐츠, 카피, SNS, 블로그, 피드 | ✍ 콘텐츠 크리에이터 | ✦ 브랜드 전략가 (톤앤매너 검수) |
| 캠페인 기획 | 캠페인, 런칭, 프로모션, 이벤트 | 🔍 + ✍ 병렬 | 📊 퍼포먼스 분석가 (KPI 설계) |
| 제안서/영업 | 제안서, 견적, 가격, 패키지 | 💰 세일즈 전략가 | 🔍 시장 조사관 (데이터 보강) |
| 보고서/리포트 | 보고서, 리포트, 성과, KPI | 📊 퍼포먼스 분석가 | — (단독) |

### 전문팀 자동 라우팅

아래 도메인 키워드가 감지되면, 범용 매핑 대신 전문팀 전체가 활성화된다.

| 도메인 키워드 | 전문팀 | 진입 스킬 |
|-------------|--------|----------|
| Meta, 페이스북 광고, 인스타 광고, ROAS | Meta 전담팀 (4명) | `/meta-campaign` |
| 네이버플레이스, 스마트플레이스, 매장, 리뷰관리 | NP 전담팀 (5명) | `/np-audit` |
| 바이럴, 숏폼, 릴스, 틱톡, Shorts | 바이럴 전담팀 (5명) | `/viral-plan` |
| 버그, 오류, 에러, 깨짐 | 디버깅팀 (3명) | `/debug` |
| 상담, 수주, 미팅 준비, 클로징 | New Business팀 (4명) | `/new-biz-prep` |

### 클라이언트별 오버라이드

- 클라이언트 폴더에 별도 라우팅 규칙이 CLAUDE.md에 명시되어 있으면 범용 규칙 대신 해당 규칙을 따른다
- 예시: 휴닉 → 콘텐츠 제작 시 `food-regulation-advisor` 법적 검수 필수 추가
- 오버라이드는 범용 규칙을 **대체**하는 것이 아니라 **확장**한다 (범용 + 추가 규칙)
